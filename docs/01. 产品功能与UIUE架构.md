# 策略研发工作台 (Strategy IDE) UI/UX 架构设计标准

## 1. 背景与当前痛点分析
当前 `/features/trading/strategy.tsx` 单文件集成了代码编辑、流式 AI 生成、AST 语法检查、沙箱推演等核心功能。由于缺乏统一的 IDE 布局规范，导致以下 UX 缺陷：
1. **视觉拥挤 (Overcrowding)**：AI 对话框、代码区、回测图表挤在同一视口，导致核心的代码编写空间受压榨，极大降低了沉浸感。
2. **工作流断层 (Workflow Disruption)**：AI 生成代码直接覆盖当前编辑器，缺乏 Diff 对比机制；报错后直接使用弹窗，缺乏持续的对话上下文。
3. **报表受限 (Cramped Reports)**：量化回测的分析（Tear Sheet、资金曲线、最大回撤图等）全挤在底部面板，无法展示高密度的数据分析。
4. **状态臃肿 (Bloated State)**：单文件内聚了近 30 个 `useState`，导致严重的 Re-render 性能开销，同时增加了后续维护和扩展的难度。

## 2. 业界标杆参考 (Benchmarks)
重构将融合以下业界顶尖产品的设计范式：
- **Cursor / GitHub Copilot Chat (AI 驱动)**：AI 作为一个智能体常驻右侧边栏（Chat），支持多轮上下文对话与代码 Diff 实时应用 (Apply)。
- **QuantConnect / VS Code (专业布局)**：标准的三栏式（左/中/右）+ 底部终端布局，支持多 Tab 切换，实现源码编写与回测分析视图的物理分离。

## 3. 核心空间拓扑布局 (Layout Topology)
采用基于 `ResizablePanelGroup` 的标准现代 IDE 布局设计：

```text
+-----------------------------------------------------------------------+
| Topbar: 策略名称 | 运行沙箱回测 | 保存 | 部署至 OMS                         |
+-------------+---------------------------------------+-----------------+
| Left Sidebar| Main Workspace Area (Tabs)            | Right Sidebar   |
| (可折叠)    |                                       | (可折叠)        |
|             | [ Tab 1: 源码 (Monaco) ] [Tab 2: 回测]| - [Tab: AI Chat]|
| - 策略草稿库|                                       |   多轮对话、修复 |
| - 因子库片段| def on_tick(self, data):              |   [Apply] 按钮  |
| - 预设历史  |     # Strategy logic here             |                 |
|             |                                       | - [Tab: 参数]   |
|             |                                       |   AST表单解析   |
|             +---------------------------------------+                 |
|             | Bottom Panel: 终端与日志 (Terminal)       |                 |
|             | > [INFO] 语法检查通过...              |                 |
+-------------+---------------------------------------+-----------------+
```

## 4. 关键交互路径设计 (UX Flows)

### 4.1 AI Vibe Coding (Text-to-Strategy) 交互
- **输入层**：顶部的 Prompt 输入框被移除，统一迁移至右侧边栏 `AI Copilot` 标签页，以对话流（Chat）形式交互。
- **思考层**：优雅地展示大模型（如 DeepSeek R1）的 `<think>` 思考过程（可折叠面板）。
- **输出层**：AI 生成代码后，不直接覆盖主编辑器。提供 `[Apply]` 按钮，利用 Monaco Diff Editor 特性展示差异，供用户 Review 后再决定是否合并。

### 4.2 沙箱推演与图表展示 (Backtest & Reports)
- **触发器**：点击顶部动作栏的【运行沙箱推演】。
- **视图切换**：运行完成后，Main Workspace 自动新建或跳转到【回测报告】Tab。
- **复用能力**：在【回测报告】Tab 中，完全复用和挂载 `/features/trading/backtest.tsx` 中已经高度成熟的全屏 Tear Sheet、多指标组合 K 线图、水下回撤图和直方图组件。

### 4.3 异常排查流 (Agentic Auto-Debug)
- **报错拦截**：沙箱运行时崩溃 (Runtime Error) 或 AST 解析失败时，将日志定向输出至底部的 `Terminal Panel`。
- **一键修复**：终端报错日志旁提供醒目的 `[✨ AI 分析与修复]` 按钮。点击后，系统自动将报错堆栈和当前代码投喂给右侧 AI Chat 侧边栏，由 Agent 完成错误分析并提供修复补丁。

## 5. 前端组件拆分架构 (Component Architecture)
将原 1000+ 行的 `strategy.tsx` 单文件按单一职责原则 (SRP) 拆分为以下目录结构：

```text
src/features/trading/strategy/
├── page.tsx                    # 页面容器，负责状态注入和 Provider 包装
├── layout/
│   ├── strategy-ide.tsx        # 核心骨架 (基于 PanelGroup)
│   ├── topbar.tsx              # 顶部全局动作栏
│   ├── left-sidebar.tsx        # 左侧边栏：策略列表管理
│   ├── right-sidebar.tsx       # 右侧容器：动态 Tab (AI 对话 / 策略参数)
│   └── bottom-terminal.tsx     # 底部输出终端：AST 日志、报错、系统状态
├── workspace/
│   ├── main-tabs.tsx           # 主工作区 Tab 视图控制器
│   ├── monaco-editor.tsx       # 隔离的 Monaco 编辑器组件 (含代码补全注册)
│   └── backtest-report.tsx     # 独立的回测报告全屏视图 (复用回测图表库)
└── stores/
    └── useStrategyStore.ts     # Zustand 状态机，接管全局 State
```

## 6. 状态管理重构 (Zustand)
全面废弃顶层堆砌的 `useState`，建立专门的 `useStrategyStore`。分为以下功能切片 (Slices)：
1. **`editorSlice`**: 管理当前编辑的纯文本源码、AST 语法树解析结果、未保存状态(`isDirty`)。
2. **`aiSlice`**: 管理多轮对话历史记录、流式生成的深度思考内容、当前的 Diff 对比状态。
3. **`backtestSlice`**: 管理表单参数配置、回测进度(Progress)、沙箱报告结果(Result)。
4. **`layoutSlice`**: 管理左右侧边栏的展开/折叠状态、当前激活的主工作区 Tab (Code/Report)。
