## 一、产品定位与信息架构

### 1.1 产品定位

QuantEdge 是一款**面向单兵量化交易者的全栈投研终端**，核心价值主张：

1. **AI 深度集成**——从选股到策略生成到风险研判，AI 贯穿全工作流
2. **毫秒级实时**——WebSocket 行情 + Canvas/WebGL 渲染，盯盘无延迟
3. **沙箱到实盘**——策略先在沙箱验证，再一键部署至 OMS 实盘
4. **三端覆盖**——Web 重型工作台 + Flutter 移动端随身监控

### 1.2 信息架构（模块导航）

```
┌────┐
│ 📊 │  市场指挥中心 (Market Hub)      — 行情/K线/盘口/新闻流
│ 🔍 │  智能选股器 (Screener)          — AI NLP 选股 + 条件过滤
│ ⚗️ │  策略实验室 (Strategy Lab)      — Monaco IDE + AI 代码生成
│ 📈 │  回测工坊 (Backtest Studio)     — Tear Sheet + 蒙特卡洛
├────┤
│ 📋 │  订单中心 (OMS)                 — 持仓/挂单/算法委托
│ 🛡️ │  风控面板 (Risk Dashboard)      — 因子暴露/归因/压测
│ 🌍 │  宏观数据中心 (Macro Hub)        — 六象限雷达/财经日历
├────┤
│ 🤖 │  AI 副驾 (Copilot)             — SSE 流式对话 + ECharts 渲染
│ 🔔 │  告警中心 (Alert Center)        — 价格/指标/策略信号告警
│ ⚙️ │  系统设置 (Settings)           — 沙箱/实盘切换 + 偏好配置
└────┘
```

```

### 1.4 设计原则

| 原则 | 说明 |
|:---|:---|
| **上下文持续** | 模块切换不卸载组件（Keep-Alive），K 线缩放、策略代码不丢失 |
| **数据密度分级** | 极密（盘口 11px mono）/ 密（AG Grid 12px）/ 标准（卡片 14px）/ 稀疏（标题 16px） |
| **键盘优先** | Cmd+K 命令面板、B/S 快速下单、Cmd+1~9 模块切换 |
| **安全常驻** | SANDBOX/LIVE 模式横幅永远可见，破坏性操作双重确认 |
| **STALE 感知** | WS 断连时数据区域自动叠加 STALE 遮罩，不展示过期数据 |

## 四、策略实验室 (Strategy Lab)

### 4.1 功能定义

AI 驱动的策略研发环境，集成 Monaco 代码编辑器、流式 AI 代码生成、AST 语法检查、沙箱推演。

### 4.2 痛点分析

原 `strategy.tsx` 单文件集成了代码编辑、AI 生成、AST 检查、沙箱推演，存在以下缺陷：
1. **视觉拥挤**：AI 对话框、代码区、回测图表挤在同一视口
2. **工作流断层**：AI 生成代码直接覆盖编辑器，缺乏 Diff 对比
3. **报表受限**：回测分析全挤在底部面板，无法展示高密度数据
4. **状态臃肿**：单文件近 30 个 `useState`，严重 Re-render 开销

### 4.3 布局设计（三栏 IDE）

```
+-----------------------------------------------------------------------+
| Topbar: 策略名称 | 运行沙箱回测 | 保存 | 部署至 OMS                         |
+-------------+---------------------------------------+-----------------+
| Left Sidebar| Main Workspace Area (Tabs)            | Right Sidebar   |
| (可折叠)    |                                       | (可折叠)        |
|             | [Tab 1: 源码 (Monaco)] [Tab 2: 回测]  | - [Tab: AI Chat]|
| - 策略草稿库|                                       |   多轮对话、修复 |
| - 因子库片段| def on_tick(self, data):              |   [Apply] 按钮  |
| - 预设历史  |     # Strategy logic here             |                 |
|             |                                       | - [Tab: 参数]   |
|             |                                       |   AST表单解析   |
|             +---------------------------------------+                 |
|             | Bottom Panel: 终端与日志 (Terminal)       |                 |
|             | > [INFO] 语法检查通过...              |                 |
+-------------+---------------------------------------+-----------------+
```

### 4.4 关键交互路径

#### 4.4.1 AI Vibe Coding (Text-to-Strategy)
- **输入层**：Prompt 输入迁移至右侧 AI Copilot Tab，以对话流形式交互
- **思考层**：展示大模型 `<think>` 思考过程（可折叠面板）
- **输出层**：AI 生成代码后不直接覆盖编辑器，提供 `[Apply]` 按钮触发 Monaco Diff Editor 展示差异

#### 4.4.2 沙箱推演与图表展示
- **触发器**：点击顶部【运行沙箱推演】
- **视图切换**：运行完成后自动跳转到【回测报告】Tab
- **复用能力**：回测报告 Tab 复用 `backtest.tsx` 中的 Tear Sheet、资金曲线、回撤图组件

#### 4.4.3 异常排查流 (Agentic Auto-Debug)
- **报错拦截**：沙箱运行时崩溃或 AST 解析失败 → 日志输出至底部 Terminal
- **一键修复**：终端报错旁提供 `[✨ AI 分析与修复]` 按钮，自动将报错堆栈和代码投喂给 AI Chat

### 4.5 组件拆分架构

```text
src/features/strategy/
├── page.tsx                    # 页面容器，状态注入和 Provider 包装
├── layout/
│   ├── strategy-ide.tsx        # 核心骨架 (ResizablePanelGroup)
│   ├── topbar.tsx              # 顶部全局动作栏
│   ├── left-sidebar.tsx        # 左侧：策略列表管理
│   ├── right-sidebar.tsx       # 右侧：动态 Tab (AI 对话 / 策略参数)
│   └── bottom-terminal.tsx     # 底部：AST 日志、报错、系统状态
├── workspace/
│   ├── main-tabs.tsx           # 主工作区 Tab 视图控制器
│   ├── monaco-editor.tsx       # Monaco 编辑器 (含代码补全注册)
│   └── backtest-report.tsx     # 回测报告全屏视图 (复用回测图表库)
└── stores/
    └── useStrategyStore.ts     # Zustand 状态机
```

### 4.6 状态管理 (Zustand Slices)

| Slice | 职责 |
|:---|:---|
| `editorSlice` | 当前编辑源码、AST 解析结果、`isDirty` 未保存状态 |
| `aiSlice` | 多轮对话历史、流式思考内容、Diff 对比状态 |
| `backtestSlice` | 表单参数配置、回测进度、沙箱报告结果 |
| `layoutSlice` | 侧边栏折叠状态、激活的 Tab (Code/Report) |

### 4.7 新增功能规划

- [ ] **版本时间线**：左侧边栏底部可视化策略版本历史，点击恢复任意版本
- [ ] **参数快速调试面板**：右侧参数滑块，拖动 MA 周期/RSI 阈值实时触发沙箱重跑
- [ ] **实盘叠加对比**：回测报告中叠加实际模拟交易收益曲线，识别过拟合
- [ ] **代码热点图**：编辑器 Gutter 区域标记每行代码的执行频率
- [ ] **策略对比模式**：并排比较两个策略版本的回测结果

┌─────────────────────────┐

│ 📋 复制代码              │
│ ─────────────────────── │
│ 📈 在新面板打开          │

```

### 11.4 面板分屏与布局持久化

- 主工作区支持左右/上下拆分（最多 2×2 四格）
- 布局状态自动持久化至 `localStorage`，刷新后恢复
- 支持"保存布局快照"并命名（如：盯盘模式、研究模式、回测模式）

### 11.5 Loading State 规范

```
数据加载中：骨架屏（Skeleton），bg-slate-800/50，animate-pulse
数据为空时：空状态插图 + 操作引导（"还没有持仓，去选股器找标的？"）

数据过期时：STALE Badge + 最后更新时间 + 重连倒计时

### 11.6 颜色语义扩展

强看跌（-1~-3%）     → text-red-400

信息提示色（ℹ️）    → text-blue-400 bg-blue-500/10
