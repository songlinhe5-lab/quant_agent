## 一、产品定位与信息架构

### 1.1 产品定位

QuantEdge 是一款**面向单兵量化交易者的全栈投研终端**，核心价值主张：

1. **AI 深度集成**——从选股到策略生成到风险研判，AI 贯穿全工作流
2. **毫秒级实时**——WebSocket 行情 + Canvas/WebGL 渲染，盯盘无延迟
3. **可信回测**——PIT 财务 + 幸存者偏差 + **数据快照版本化**，回测结果可复现（`DQ-*` · `docs/19`）
4. **沙箱 → 纸面 → 实盘**——单次沙箱推演 → 常驻纸面组合验证 → 版本化部署实盘（`PT-*` · `docs/17`）
5. **三端覆盖**——Web 重型工作台 + Flutter 移动端随身监控（告警推送为核心场景）

### 1.4 设计原则

| 原则 | 说明 |
|:---|:---|
| **上下文持续** | 模块切换不卸载组件（Keep-Alive），K 线缩放、策略代码不丢失 |
| **数据密度分级** | 极密（盘口 11px mono）/ 密（AG Grid 12px）/ 标准（卡片 14px）/ 稀疏（标题 16px） |
| **键盘优先** | Cmd+K 命令面板、B/S 快速下单、Cmd+1~9 模块切换 |
| **三模式安全常驻** | SANDBOX / PAPER / LIVE 横幅永远可见（§1.6）；破坏性操作双重确认 |
| **STALE 感知** | WS 断连时数据区域自动叠加 STALE 遮罩，不展示过期数据 |
| **可复现优先** | 正式回测默认绑定数据快照 + 策略版本（`docs/19` · `docs/16`），非 live 数据 |

### 1.5 核心用户旅程（V2.2 新增）

> 产品主路径；各步骤对应模块与架构 doc 见下表。未列出的「单次沙箱」为策略实验室内循环，不替代纸面长期验证。

```
路径 A · 投研闭环（主路径）
  选股器 NLP/条件筛选 → [→回测] 策略实验室沙箱 → 保存策略版本
       → 部署为 Paper Bot → 纸面组合 N 日绩效 → deploy 实盘（带纸面检查点 · docs/17）

路径 B · 选股直达组合回测
  选股结果 [→回测] → 回测工坊组合 Tear Sheet（`QUANT-02`，待建）

路径 C · 无人值守盯盘
  行情页右键 [设价格告警] → 告警规则 CRUD → 服务端引擎触发 → 多通道推送（docs/18）
       → 告警中心历史 / 移动端推送（ALERT-04/05）
```

|:---|:---|:---|:---|

| 告警 | 离线触达 | §十 + 顶栏 🔔 | `ALERT-*` · docs/18 |

### 1.6 运行模式规范（V2.2 新增 · 三模式）

| 模式 | 语义 | 顶栏标识 | 下单/撮合 | 典型场景 |
|:---|:---|:---|:---|:---|
| **SANDBOX** | 单次推演，无持久账本 | 🟡 橙色横幅 | Futu 模拟盘或纯内存（见 OMS 实现） | 策略 IDE 点「运行沙箱」 |
| **PAPER** | 常驻纸面组合，虚拟账本 SSOT | 🟠 橙色横幅 + 「PAPER」 | SimBroker 记账（docs/17） | Paper Bot 长期运行 |
| **LIVE** | 真实资金 | 🔴 红色脉冲横幅，不可关闭 | Futu 实盘 + `REAL_TRADE_EXECUTE` 双锁 | 生产交易 |

**产品规则**：

- 模式切换仅能在 **Settings** 或 OMS 顶栏（需二次确认）；PAPER ↔ LIVE 切换须展示当前纸面组合摘要（运行天数 / Sharpe / TE · docs/17）
- 用户不可从 SANDBOX 单次回测直接跳 LIVE；必须经过「保存版本 → Paper 或显式风险确认」
- Kill Switch 触发后：全模式冻结下单 UI，P0 告警（docs/18）

### 3.3 核心工作流

```
用户输入自然语言 → AI 解析为 DSL → 条件构建器可编辑 → 执行筛选 → 结果表格
                                                                    ↓
                                                            ┌───────┴───────┐
                                                            ↓               ↓
                                                        →回测工坊      →告警监控
```

### 3.4 核心交互

| 交互 | 触发方式 | 行为 |
|:---|:---|:---|
| AI 解析 | 输入自然语言后点击"⚡AI 解析" | LLM 将自然语言转为筛选 DSL |
| 条件增删 | 条件构建器内操作 | 实时更新结果表格 |
| 表头排序 | 点击 AG Grid 表头 | 按 PE/ROE/涨跌幅等排序 |
| 行展开预览 | 点击结果行 | 展开迷你 K 线 + 关键指标 + AI 短评 |
| 一键回测 | 点击"→回测" | 将选中标的生成等权重组合跳转至回测工坊 |
| 模板加载 | 点击"模板库▾" | 加载经典选股策略模板（价值低估/动量突破/高股息） |

### 4.1 功能定义

AI 驱动的策略研发环境，集成 Monaco 代码编辑器、流式 AI 代码生成、AST 语法检查、沙箱推演。

### 4.2 痛点分析（历史 · 2026-06）

> ⚠️ **V2.2 勘误**：下列痛点针对 2026-06 单文件 `strategy.tsx` 时代。截至 2026-07，三栏骨架已拆分（`features/strategy/` 多文件），**仍待完成**：4 Slice Store、Diff 工作流、PG 版本（docs/16）。

原 `strategy.tsx` 单文件集成了代码编辑、AI 生成、AST 检查、沙箱推演，存在以下缺陷：
1. **视觉拥挤**：AI 对话框、代码区、回测图表挤在同一视口
2. **工作流断层**：AI 生成代码直接覆盖编辑器，缺乏 Diff 对比
3. **报表受限**：回测分析全挤在底部面板，无法展示高密度数据
4. **状态臃肿**：单文件近 30 个 `useState`，严重 Re-render 开销

### 4.4 关键交互路径

#### 4.4.1 AI Vibe Coding (Text-to-Strategy)
- **输入层**：Prompt 输入迁移至右侧 AI Copilot Tab，以对话流形式交互
- **思考层**：展示大模型 `<think>` 思考过程（可折叠面板）
- **输出层**：AI 生成代码后不直接覆盖编辑器，提供 `[Apply]` 按钮触发 Monaco Diff Editor 展示差异

#### 4.4.2 沙箱推演与图表展示
- **触发器**：点击顶部【运行沙箱推演】
- **视图切换**：运行完成后自动跳转到【回测报告】Tab
- **复用能力**：回测报告 Tab 复用 `backtest.tsx` 中的 Tear Sheet、资金曲线、回撤图组件

#### 4.4.3 异常排查流 (Agentic Auto-Debug)
- **报错拦截**：沙箱运行时崩溃或 AST 解析失败 → 日志输出至底部 Terminal
- **一键修复**：终端报错旁提供 `[✨ AI 分析与修复]` 按钮，自动将报错堆栈和代码投喂给 AI Chat
