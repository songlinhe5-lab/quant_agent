# 性能测试与基准规范 (Performance Testing & Benchmarks)

> **原则**：不猜测性能，只测量性能。每次优化前必须有基准数据，每次优化后必须有对比数据。

### 2.1 单元性能测试（pytest-benchmark）

```python
# backend/tests/benchmarks/test_screener_benchmark.py

import pytest
from backend.services.screener_service import ScreenerService

@pytest.mark.benchmark(group="screener")
def test_screener_filter_1000_stocks(benchmark, mock_stock_pool_1000):
    """基准：1000只股票，3个条件过滤，目标 < 100ms"""
    service = ScreenerService.__new__(ScreenerService)  # 跳过 __init__
    result = benchmark(service._apply_filters, mock_stock_pool_1000, conditions={
        "max_pe": 15.0, "min_roe": 0.15, "min_market_cap": 1e9
    })
    assert len(result) > 0
    # benchmark 自动统计 min/max/mean/stddev/ops_per_sec

@pytest.mark.benchmark(group="backtest")
def test_backtest_1year_daily(benchmark, mock_ohlcv_1year):
    """基准：1年日线单标的向量化回测，目标 < 500ms"""
    from backend.core.backtest_engine import VectorizedBacktest
    engine = VectorizedBacktest()
    result = benchmark(engine.run, data=mock_ohlcv_1year, strategy_params={"ma": 20})
    assert result.sharpe_ratio is not None
```

**运行方式**：
```bash
# 运行所有 benchmark，生成 JSON 报告
pytest backend/tests/benchmarks/ --benchmark-json=reports/benchmark_backend.json

# 与基准对比（CI 中使用，超过 20% 视为性能退化）
pytest-benchmark compare reports/benchmark_baseline.json reports/benchmark_backend.json
```

### 2.2 接口压测（Locust）

```python
# backend/tests/load/locustfile.py

from locust import HttpUser, task, between

class QuantEdgeUser(HttpUser):
    wait_time = between(0.5, 2.0)

    @task(weight=5)
    def get_quote(self):
        """高频：行情快照查询"""
        self.client.get("/api/v1/market/quote/AAPL", name="/api/v1/market/quote/{symbol}")

    @task(weight=2)
    def run_screener(self):
        """中频：选股器"""
        self.client.post("/api/v1/screener", json={
            "market": "HK", "max_pe": 15, "min_roe": 0.12
        }, name="/api/v1/screener")

    @task(weight=1)
    def get_kline_history(self):
        """低频：K线历史"""
        self.client.get("/api/v1/market/kline/AAPL?period=1d&count=250",
                        name="/api/v1/market/kline/{symbol}")
```

**运行方式**：

### 2.3 WebSocket 吞吐测试

```python
# backend/tests/benchmarks/test_ws_benchmark.py

async def test_websocket_throughput_50_clients():
    """50个并发 WS 客户端同时接收 Tick，测量吞吐和延迟"""
    clients = [await create_ws_client() for _ in range(50)]
    start = time.perf_counter()
    tick_counts = await asyncio.gather(*[
        collect_ticks(client, duration_sec=10) for client in clients
    ])
    elapsed = time.perf_counter() - start

    total_ticks = sum(tick_counts)
    ticks_per_sec = total_ticks / elapsed

    logger.info("ws_throughput_test",
                clients=50, total_ticks=total_ticks,
                ticks_per_sec=round(ticks_per_sec, 1), elapsed_sec=round(elapsed, 2))

    assert ticks_per_sec >= 1000, f"吞吐不达标: {ticks_per_sec:.0f} Tick/s < 1000"
```

## 三、前端性能测试体系

### 3.1 Tick 渲染延迟测试

```typescript
// frontend/tests/perf/tick-render.perf.ts

describe("Tick 渲染性能", () => {
  it("单个 Tick 更新耗时应 ≤ 16ms (60fps)", () => {
    const chart = mountLightweightChart()
    const ticks = generateMockTicks(100)

    const times: number[] = []
    for (const tick of ticks) {
      const start = performance.now()
      chart.update(tick)           // 直接调用图表 API，不经过 React
      times.push(performance.now() - start)
    }

    const p99 = percentile(times, 99)
    expect(p99).toBeLessThan(16)  // 60fps 帧预算
    console.log(`Tick 渲染 P99: ${p99.toFixed(2)}ms`)
  })

  it("5000行 AG Grid 初始渲染应 ≤ 200ms", async () => {
    const start = performance.now()
    render(<ScreenerResultsTable data={generateMockStocks(5000)} />)
    await waitForGridReady()
    const elapsed = performance.now() - start
    expect(elapsed).toBeLessThan(200)
  })
})
```

## 五、CI 性能回归检测

```yaml
# .github/workflows/performance.yml
name: Performance Regression Check

on: [push]

jobs:
  backend-benchmark:
    runs-on: ubuntu-latest
    steps:
      - name: Run benchmarks
        run: pytest backend/tests/benchmarks/ --benchmark-json=current.json

      - name: Compare with baseline
        run: |
          pytest-benchmark compare baseline.json current.json \
            --fail-faster \
            --histogram \
            --sort=name

      - name: Fail if regression > 20%
        run: |
          python scripts/check_regression.py \
            --baseline baseline.json \
            --current current.json \
            --threshold 0.20     # 超过 20% 视为性能退化，CI 失败
```
