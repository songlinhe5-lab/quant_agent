## 一、设计原则（规范性）

| # | 原则 | 硬性要求 |
|:---|:---|:---|
| 1 | **依赖只向内** | Domain 不 import FastAPI / Redis 客户端实现细节以外的 Adapter；Router 不 import 具体 `*_service` 数据源类 |
| 2 | **接口稳定** | 跨模块只依赖 **Port（Protocol / ABC）** 或版本化 Pydantic Schema；禁止依赖对方内部私有函数 |
| 3 | **不交叉引用** | 同级 Application 之间不互相 import；共享能力下沉 Domain Port 或 Application 编排 |
| 4 | **单一装配点** | 具体类实例化只允许在 Composition Root（`main.py` / `worker.py` / 测试 fixtures） |
| 5 | **可插拔** | 数据源 / 采集器 / 告警通道 / 策略实现经 Registry 发现；新增实现不改调用方 |
| 6 | **分级热加载** | 策略可热加载；Alert Adapter 可热注册；数据源 SDK **仅冷启动**（见 §五） |
| 7 | **模块稳定** | L0 路径改动绑定 TODO 任务 + PCE Confirm（`docs/02` §2.3 / §7.6） |

### 2.2 部署拓扑（与 AGENTS §9 对齐）

```
加州主节点 VPS
  API + Worker + Redis + PostgreSQL
  Futu / YFinance / Finnhub / FRED（本地或 Router）
  DataSourceRegistry + ServiceRegistry（节点发现，docs/14）

北京辅助节点
  仅 AKShare 采集 → Redis 中继 / HMAC 子服务
  不承载主 API

Hermes Agent
  仅经 /internal/v1/*（HMAC）调 Application，禁止直连外部行情 API
```

> **废止 V5.0 表述**：「无分布式路由/服务注册」。Registry 与 Router 是合法架构组件；禁止的是 **业务层绕过 Registry 直连外部库**。

### 2.3 禁止 import 矩阵（规范性）

| From ↓ \ To → | Domain | Application | Adapter 具体类 | routers | workers |
|:---|:---:|:---:|:---:|:---:|:---:|
| **Domain** | — | ❌ | ❌ | ❌ | ❌ |
| **Application** | ✅ Port | ⚠️ 仅编排调用，禁循环 | ❌ | ❌ | ❌ |
| **Adapter** | ✅ 实现 Port | ❌ | 同族可 | ❌ | 采集可调用同族 |
| **routers** | ❌ | ✅ | ❌ | — | ❌ |
| **workers** | ❌ | ✅ 或 Port | ✅ 采集路径 | ❌ | — |
| **Composition Root** | ✅ | ✅ | ✅ 装配 | ✅ 挂载 | ✅ 启停 |

**当前违规（债务，收口见 §八）**：`routers/market.py`、`strategy.py`、`macro.py`、`backtest.py` 等直接 `import futu_service / yf_service / …`。V5.1 起新代码 **禁止新增** 此类 import；存量按 `BE-ARCH-01` 收敛。

### 2.4 模块冻结分级（映射 docs/02 §2.3）

| 级别 | 路径（示例） | 规则 |
|:---|:---|:---|
| 🔴 L0 | `backend/core/{config,database,security,redis_client,response}*` · `services/datasource/registry*` · `workers/collector_registry.py` · `backend/engine/`（落地后） | 无 TODO 任务禁止改；须 PCE Confirm |

## 三、稳定接口（Ports）与契约

### 3.1 跨模块只暴露 Port / Schema

```python
# 示意：Domain Port（稳定，版本化）
from typing import Protocol, runtime_checkable

@runtime_checkable
class QuotePort(Protocol):
    async def get_quote(self, symbol: str) -> "QuoteDTO": ...
    async def get_history(self, symbol: str, period: str, **kw) -> "HistoryDTO": ...

@runtime_checkable
class BrokerPort(Protocol):
    async def place_order(self, req: "OrderRequest") -> "OrderResult": ...
    async def cancel_order(self, order_id: str) -> "OrderResult": ...
```

- **DTO / 请求响应**：Pydantic v2，放 `backend/schemas/` 或 `domain/dto/`；破坏性变更升 API 版本（`docs/10`）
- **禁止**：Application 依赖 `FutuService._conn`、私有 Handler、未导出函数
- **数据源**：一律 `DataSourceInterface.fetch(action, params) -> Result`（`docs/14` §二）；业务禁止 `yf.Ticker()` / `futu_api` 直调

### 3.2 三通道 API 隔离（保留）

| 通道 | 前缀 | 鉴权 | 调用方 |
|:---|:---|:---|:---|
| 公开 REST | `/api/v1/` | JWT | Web / App |
| WebSocket | `/ws/v1/` | JWT 握手 | Web / App |
| SSE | `/sse/v1/` | JWT | Web / App |
| 内部 Tool | `/internal/v1/` | HMAC-SHA256 | Hermes Tools |

端点清单 **SSOT = `docs/10`**。本文不维护全量路径表（避免双轨漂移）。

### 3.3 Hermes 边界

```
✅ Tool → /internal/v1/* → Application → Port → Adapter/Cache
❌ Tool 直连 Futu/YFinance/PG
❌ Tool 调用 /api/v1/*（用户 JWT 通道）
```

## 四、可插拔组件模型

### 4.1 统一插件生命周期

所有可插拔组件实现：

```python
class PluginInfo(Protocol):
    name: str
    version: str              # semver；破坏性变更升 major
    capabilities: frozenset[str]

class Lifecycle(Protocol):
    async def start(self, ctx: "AppContext") -> None: ...
    async def health(self) -> "HealthInfo": ...
    async def stop(self) -> None: ...
```

| 插件族 | Registry | 启用方式 | 详规 |
|:---|:---|:---|:---|
| 采集器 | `workers/collector_registry.py` | `COLLECTOR_*` | 本文 §4.2 |
| 数据源 | DataSource Registry（docs/14） | `DATASOURCE_{NAME}_*` | docs/14 |
| 告警通道 | AlertDispatcher adapters | 配置 / 注册表 | docs/18 |
| 策略实现 | `strategies/` + version_id | deploy API | docs/15 · 16 |

**可扩展规则**：新增实现 = 新 Adapter 文件 + Registry 注册 + env；**禁止**修改调用方 if/else 丛林（OCP 仅在此场景优于 YAGNI）。

### 4.2 采集器插件（现状与目标）

**现状（BE-ARCH-03 ✅ 2026-07-13）**：`CollectorDef.factory` + `workers/collectors/<name>.py`；`start_collector_daemons` 只遍历 factory 并 `create_task`，**零具体服务 import**。

```
CollectorDef + factory: Callable[[], Awaitable[Sequence[Coroutine]]]
start_collector_daemons 只遍历 factory，零具体服务 import
新增采集器 = collectors/<name>.py + COLLECTORS 注册 + COLLECTOR_* env
```

### 4.3 数据源插件

- 每个源实现 `DataSourceInterface`（`services/datasource/protocol.py`）
- **双 Registry（BE-ARCH-04 ✅）**：
  - `RateLimitRegistry`（`registry.py` / `rate_limit_registry`）— 仅 Throttler + Analyzer（docs/14 §十二）
  - `DataSourceRegistry`（`source_registry.py` / `datasource_registry`）— 源实例生命周期 + **`fetch` 主路径**（docs/14 §5）
- 主路径只经 `datasource_registry.fetch` → Interface；限流退避期不打真实请求
- YFinance 已挂 Legacy Adapter；Futu/AKShare/Finnhub 仍可经 QuotePort Legacy Gateway，按源渐进迁入

## 五、热加载分级（规范性）

| 级别 | 对象 | 策略 | 禁止 |
|:---|:---|:---|:---|
| 🟢 **热加载** | 策略脚本（`strategies/live/`） | `importlib` 按 **strategy_version_id** 加载；沙箱 AST（docs/16）；失败不影响其他 Bot | `importlib.reload` 核心单例；无版本 ID 的裸文件覆盖实盘 |
| 🟠 **热注册** | Alert Channel Adapter、只读 Tool 扩展 | 进程内 Registry `register()`；旧实例可 drain | 热 unload 仍持有连接的 Adapter 而不 `stop()` |
| 🔴 **冷启动** | 数据源 SDK、DB 引擎、Redis 客户端、Futu OpenD 连接 | 仅进程启动 / 滚动重启 | 运行时 unload 数据源模块 |

**Bot 热加载现状**：`bot_runtime` 已用 `importlib.util` 加载策略文件——保留，但必须升级到 **统一 Strategy Port（docs/15）+ version_id（docs/16）**，禁止三套策略契约并存长期化。

## 七、横切能力（压缩保留）

### 7.1 认证与安全

- JWT 双 Token（access 短时 + refresh HttpOnly）；内部 HMAC（Hermes）
- 凭证仅环境变量 / Docker Secret；审计 append-only（`audit_logs`）
- 详规与实现：`backend/core/security.py` · `middleware` · SEC-* 任务

### 7.2 行情流水线（摘要）

```
Futu Tick → workers/quote_publisher
  → Redis（quote / kline:live / kline:hist）
  → Pub/Sub → WebSocket Gateway → 前端
  → 异步落 Parquet / DuckDB（温冷）
```

订阅配额超限 → 降级轮询快照 + UI STALE（L0 铁律）。Redis Key 规范见 §7.4。

### 7.3 缓存分层

| 层 | 存储 | 延迟 | 用途 |
|:---|:---|:---|:---|
| 热 | Redis | ms | 行情快照、活跃 K 线、会话 |
| 温/冷 | DuckDB + Parquet | 百 ms~秒 | 历史 K 线、回测 |
| 源 | 外部 API | 不定 | 仅 Adapter / workers |

快照不可变层见 **docs/19**（正式回测默认 `latest_published`，禁止无意读 live）。

### 7.4 Redis Key 命名（保留）

```
格式：{业务域}:{资源}:{标识符}:{可选维度}

quote:{symbol}
kline:live:{symbol}:{period}
kline:hist:{symbol}:{period}
quant:cache:{action}:{ticker}
quant:oms:bot:{bot_id}:*
quant:alerts:*                    ← docs/18
quant:paper:*                     ← docs/17
session:chat:{session_id}
rate_limit:{ip}:{endpoint}
```

### 7.5 存储原则

| 存储 | 职责 |
|:---|:---|
| PostgreSQL | 订单、用户、策略版本、告警规则、审计、纸面账本、pgvector |
| Redis | 总线、缓存、限流、Bot 元数据 |
| Parquet 湖 | K 线 / 因子；**版本化快照** docs/19 |

表演进以 Alembic 为准；本文不维护过时 DDL 清单。

### 7.6 并发与背压

- CPU 重活：`asyncio.to_thread` / 进程池（Router 内禁止同步阻塞）
- WS 背压：慢客户端丢弃或断开；Tick 不进业务线程无限队列
- 限流：数据源限流与熔断解耦（docs/14 §十二 · RL-*）
