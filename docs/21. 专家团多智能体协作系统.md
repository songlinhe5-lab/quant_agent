# 21. 专家团多智能体协作系统

> 版本: v1.0 | 状态: Phase 1 已交付 | 测试: 38/38 通过

## 一、系统定位

独立于 Hermes Agent 的多智能体辩论引擎。

| 维度 | Hermes Agent | Expert Team |
|:---|:---|:---|
| 模式 | 1 Agent × N Tools × 1 结论 | N Experts × 共享数据 × 辩论 × 1 首席收敛报告 |
| 适用 | 简单查询、快速回答 | 深度研判、多维决策 |
| 模型 | STANDARD 单模型 | 专家 STANDARD + 首席 FLAGSHIP |

## 二、三轮混合协议

```
Round 1 - 独立研判 (Parallel)
  各专家仅看到 [用户问题 + 共享数据包]
  独立输出: 观点 + 置信度(0-100) + 关键依据
  互不可见，防止锚定效应

Round 2 - 交叉辩论 (Adversarial)
  各专家看到 [自己的 Round1 输出 + 其他专家的 Round1 摘要]
  任务: 找出对方逻辑漏洞 / 补充被忽略的风险 / 修正自己的判断
  输出: 修正后观点 + 对其他专家的质疑点 + 置信度变化

Round 3 - 首席收敛 (Synthesis)
  首席分析师看到 [所有 Round1 + Round2 全文]
  任务: 识别共识区 / 分歧区 / 各方最强论据
  输出: 最终研判报告 + 概率评估 + 风险提示 + 少数派意见保留
```

```

│   ├── industry_analyst       行业分析师     格局/空间/竞争
│   ├── sentiment_analyst      情绪分析师     资金流/持仓/散户行为
│   └── news_analyst           新闻分析师     信息质量/定价充分性/叙事转变
├── 🔬 研究员团队 (researcher) - 2位
│   ├── industry_researcher    产业研究员     产业链/护城河/技术路线
│   └── quant_researcher       量化研究员     因子/回测/统计建模

│   └── trade_executor         交易执行专家   择时/仓位/执行算法/止损

│   ├── risk_officer           风控官 [bearish]  尾部风险/黑天鹅
│   └── portfolio_risk_manager 组合风控经理 [bearish]  VaR/压力测试/对冲
├── 👔 管理层 (management) - 1位
│   └── chief_investment_officer 首席投资官   资产配置/投委会决策
```

## 八、关键设计决策

| 决策 | 选择 | 理由 |
|:---|:---|:---|
| 专家间通信 | 不共享 messages 历史 | 防止锚定效应，保证独立性 |
| 数据采集 | 一次采集，全员复用 | 避免重复调用外部 API |
| Round2 上下文 | 自己全文 + 他人摘要 | 控制 token 消耗 |
| 模型分级 | 专家 STANDARD / 首席 FLAGSHIP | 成本与质量平衡 |
| 超时策略 | 单专家 60s / 整轮 180s | 熔断后跳过，不阻塞全局 |
| 风控官 bias | 天然 bearish | 职责要求发现风险 |
| 会话存储 | 内存 (后续迁移 Redis) | Phase 1 简化 |

## 九、复用与隔离

| 组件 | 复用方式 |
|:---|:---|
| LLMRouter (llm_service) | 直接复用，专家用 STANDARD，首席用 FLAGSHIP |
| ToolRegistry | 复用，data_collector 通过 registry 调用工具采集共享数据 |
| Redis | 复用 (Phase 2 迁移会话存储) |
| Hermes Agent | 不复用 (Hermes 是单 Agent ReAct，Expert Team 是多 Agent 辩论) |

## 十一、使用者与中间产物复用

### 系统使用者

| 角色 | 使用方式 | 消费什么 |
|:---|:---|:---|
| 投资者/交易员 | 发起研判、阅读最终报告 | 首席收敛报告 + 概率评估 + 操作建议 |
| Hermes Agent | 复杂问题自动升级至专家团 | 最终报告作为 Agent 回答的增强 |
| AI 分析模式（全屏对话） | 用户触发“深度研判”指令 | 完整辩论过程 + 各专家卡片 |
| 前端专家团面板 | 可视化辩论过程 | 实时 SSE 事件流 |

### 中间产物复用

辩论过程中每个专家的独立输出，本身就有独立价值：

| 中间产物 | 可复用方 | 场景 |
|:---|:---|:---|
| 新闻分析师 → 信息质量评分 | 盯盘模式/推送过滤 | 只推 A 级新闻，过滤噪音 |
| 情绪分析师 → 情绪温度 | 监控告警模块 | 情绪极值触发告警 |
| 风控官 → 风险清单 | 持仓风控面板 | 实时风险看板 |
| 技术面分析师 → 关键价位 | 告警/自动交易 | 支撑压力位触发通知 |
| 量化研究员 → 因子信号 | 选股器/回测 | 信号输入 |
| 交易执行专家 → 交易计划 | OMS/纸面组合 | 一键下单建议 |
| 估值专家 → 估值区间 | 行情页标注 | 当前价 vs 合理区间 |
| Round 2 质疑点 | 用户学习 | 理解多空逻辑碰撞 |

### 价值分层

```
最终报告 → 给用户做决策
中间产物 → 给系统各模块做增强（告警/过滤/标注/信号）
辩论过程 → 给用户做认知升级（看到完整的多空博弈）
```
