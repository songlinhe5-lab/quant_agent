## 一、问题定义与目标

### 1.1 现状割裂（2026-07-12 代码摸底结论）

策略代码在回测与实盘是**三套互不兼容的契约**：

| 维度 | 回测·矢量化 | 回测·事件驱动 | 实盘 Bot |
|:---|:---|:---|:---|
| 承载代码 | `backend/backtest/`（VectorBT 路径） | `EventDrivenBacktestEngine`（手动 for 循环） | `backend/services/bot_runtime.py` |
| 策略接口 | `_calculate_indicators()` + `_generate_signals()` → `df["signal"]` 列 | `on_bar(window_df) → dict{action,...}` | `on_tick(quote, params) → str \| None` |
| 数据输入 | 完整 DataFrame（含未来数据，靠约定防前视） | 截至当前 bar 的窗口 DataFrame | Futu quote 单次快照（直连，不走行情总线） |
| 下单执行 | 引擎内撮合 / VectorBT 模拟 | 引擎内撮合（限价/止损） | **不下单**，仅写日志 |
| 持仓/资金 | 引擎内部状态 | 引擎内部状态 | 不读取（OMS 持仓与 Bot 无连接） |

衍生问题（同构设计必须一并收敛）：

1. **Bot 执行链断裂**：`live/` 下 3 个部署策略均无 `on_tick`，部署后永远不产生信号；即使产生信号也没有任何下单路径。
2. **DQ 模块零接线**：`financial_pit.py`（PIT 财务）与 `survivorship_bias.py`（动态标的池）已完成但回测引擎完全未调用——前视偏差与幸存者偏差依旧存在。
3. **数据路径不统一**：回测走 `KlineWarehouse` Parquet / 遗留 `parquet_db`，实盘直连 `futu_service.get_quote()`，行情总线 `quant:quotes:stream`（Protobuf）无人消费。
4. **订单状态枚举不一致**：`schemas/domain.py` 的 `OrderStatus.PARTIAL` vs OMS 实际写库的 `"PARTIALLY_FILLED"`。
5. **安全锁未接线**：`REAL_TRADE_EXECUTE` 仅存在于 `core/config.py`，Bot 与回测均未检查。
6. **BT-02 无代码**：无 `BacktestReport` 表、无 code hash、蒙特卡洛无固定种子。

### 1.2 目标（Definition of Done）

- **策略只写一次**：同一份策略类，不改一行代码，可跑「回测 → 纸面（PT-01）→ 实盘」三种模式，行为差异只来自 Driver 注入。
- **统一时间语义**：策略只能通过 `ctx.now` 感知时间，只能读取「截至 ctx.now 已发生/已公布」的数据——PIT 财务与动态标的池在 Context 层强制接线，前视偏差在架构上被消灭而非靠自觉。
- **统一执行语义**：策略产出 `OrderIntent`，由 ExecutionGateway 路由到模拟撮合（回测）或 OMS + Futu（实盘），实盘前置 `REAL_TRADE_EXECUTE` + `trading_mode` 双重安全锁。
- **VectorBT 快慢双轨同构**（QUANT-01）：事件驱动是**语义权威**，矢量化是**加速通道**（供 BT-03/05/06 的批量寻优使用），两轨结果一致性由「同构校验器」契约测试保证。
- **为 BT-02~06 预留插座**：run_id / code_hash / data_snapshot_id / random_seed 在本设计中即定义为 Context 一等公民。

### 1.3 非目标（明确不做，防过度设计）

- ❌ 不做多资产组合级撮合引擎（当前策略均为单标的/双标的，YAGNI；接口层预留 symbol 维度即可）
- ❌ 不做 Tick 级回测（数据湖只有 K 线；`on_tick` 接口保留但回测模式不驱动）
- ❌ 不引入 ZeroMQ/msgpack 新链路（现有代码无 ZMQ 落地，实盘行情消费沿用 Redis Pub/Sub + Protobuf）
- ❌ 不重写 OMS（复用 `oms_service` + `futu_service.place_order`，只加适配层）
- ❌ 不做策略进程级隔离（Bot 仍以 asyncio.Task 承载；重计算用 `asyncio.to_thread` 隔离即可）

## 二、总体架构

```
                        ┌─────────────────────────────────────────┐
                        │        Strategy（用户策略，只写一次）      │
                        │  on_init / on_bar / on_order_update      │
                        │  [可选] signals() 矢量化快路径声明          │
                        └───────────────────┬─────────────────────┘
                                            │ 只依赖 ↓（唯一 API 面）
                        ┌───────────────────▼─────────────────────┐
                        │        StrategyContext（Protocol）       │
                        │  时间: now / mode / run_id               │
                        │  数据: history() / quote() / financial() │
                        │        universe()                        │
                        │  账户: position() / cash / equity        │
                        │  执行: order() / cancel() / open_orders()│
                        │  日志: log()                             │
                        └──────┬──────────────────────┬───────────┘
                               │                      │
            ┌──────────────────▼──────┐    ┌──────────▼──────────────────┐
            │     BacktestDriver      │    │         LiveDriver          │
            │  SimClock（bar 推进）    │    │  WallClock                  │
            │  KlineWarehouse 回放    │    │  Redis quant:quotes:stream  │
            │  + PIT/Universe as-of   │    │  订阅 + K线聚合(降级轮询)      │
            │  SimBroker 撮合          │    │  OMS 持仓/资金真实态          │
            │  [快路径] VectorBT 引擎  │    │  60s→事件驱动 loop           │
            └──────────┬──────────────┘    └──────────┬──────────────────┘
                       │                              │
            ┌──────────▼──────────────────────────────▼──────────┐
            │            ExecutionGateway（统一订单出口）           │
            │  OrderIntent → ①SimBroker（回测/纸面模拟撮合）        │
            │             → ②OmsExecutionAdapter（实盘）           │
            │               REAL_TRADE_EXECUTE + trading_mode 双锁 │
            │               oms_service.create_order              │
            │               + futu place_order（to_thread）        │
            └─────────────────────────────────────────────────────┘
```

**核心原则**：策略代码的 import 面里**只允许出现** `Strategy` 基类与 `StrategyContext` 类型——策略永远不知道自己跑在回测还是实盘。所有环境差异（数据来源、时钟、撮合、持仓）被 Driver 吸收。

## 三、核心契约设计（Pydantic 先行）

### 3.1 数据契约（`contracts.py`）

```python
class Bar(BaseModel):
    symbol: str
    dt: datetime                  # bar 收盘时间（时区感知，UTC）
    open: float; high: float; low: float; close: float
    volume: float
    ktype: str                    # K_DAY / K_60M / ...

class QuoteSnapshot(BaseModel):
    symbol: str
    dt: datetime
    price: float
    bid: float | None = None
    ask: float | None = None
    stale: bool = False           # 数据降级标识（对齐 STALE 规范）

class OrderIntent(BaseModel):
    """策略产出的下单意图——唯一的执行入口契约"""
    symbol: str
    side: Literal["BUY", "SELL"]
    qty: int = Field(gt=0)
    order_type: Literal["MARKET", "LIMIT"] = "MARKET"
    limit_price: float | None = None
    stop_loss: float | None = None
    tag: str | None = None        # 策略自定义标签，贯穿至成交回报

class OrderUpdate(BaseModel):
    """订单回报（状态统一后的规范化枚举，一处修正 PARTIAL 不一致问题）"""
    order_id: str
    intent_tag: str | None
    status: OrderStatus           # 统一使用 schemas/domain.py 枚举，OMS 写库值同步修正
    filled_qty: int
    avg_fill_price: float | None

class RunManifest(BaseModel):
    """一次运行的可复现性指纹（BT-02 插座，本期只定义并填充，不做持久化）"""
    run_id: str                   # uuid
    mode: Literal["backtest", "paper", "live"]
    code_hash: str                # 策略源码 sha256
    params: dict
    data_snapshot_id: str | None  # DQ-03 就绪前为 None
    random_seed: int | None
    engine_version: str
```

### 3.2 策略基类（`strategy.py`）

```python
class Strategy(ABC):
    """用户策略唯一基类。禁止在策略内 import 任何 service / redis / futu。"""

    params: dict = {}                       # 构造时注入，网格寻优的变量面

    def on_init(self, ctx: StrategyContext) -> None: ...
        # 声明订阅标的、预热窗口长度（ctx.subscribe(symbols, warmup=60)）

    @abstractmethod
    def on_bar(self, ctx: StrategyContext, bar: Bar) -> None: ...
        # 唯一必须实现的方法。逐 bar 驱动，回测与实盘语义完全一致。

    def on_order_update(self, ctx: StrategyContext, update: OrderUpdate) -> None: ...
        # 可选。成交/拒单回报。

    def on_stop(self, ctx: StrategyContext) -> None: ...
        # 可选。清理钩子（实盘停止 / 回测结束）。

    # ---- QUANT-01 矢量化快路径（可选声明） ----
    @classmethod
    def signals(cls, df: pd.DataFrame, params: dict) -> pd.Series | None:
        """纯函数：完整 OHLCV → 信号列（1/-1/0）。
        返回 None 表示本策略不支持矢量化，只能走事件驱动。
        实现者必须保证与 on_bar 语义等价（由同构校验器验证）。"""
        return None
```

**与旧接口的关系**：

| 旧契约 | 新归宿 |
|:---|:---|
| `_calculate_indicators` + `_generate_signals` | 合并为 `signals()` 类方法（纯函数化，去 self.df 可变状态） |
| 回测 `on_bar(window_df) → dict` | `on_bar(ctx, bar)` + `ctx.order(...)`（窗口数据改为按需 `ctx.history()` 拉取） |
| 实盘 `on_tick(quote, params) → str` | 废弃。实盘也走 `on_bar`（LiveDriver 负责 tick→bar 聚合）；`on_tick` 钩子保留签名但 V1 不驱动 |

### 3.3 StrategyContext（`context.py`）

```python
class StrategyContext(Protocol):
    # ---- 元信息 ----
    now: datetime                 # 唯一合法时间源。回测=当前 bar 时间，实盘=墙钟
    mode: Literal["backtest", "paper", "live"]
    run_id: str

    # ---- 数据面（全部隐式 as-of ctx.now，架构级防前视） ----
    def history(self, symbol: str, n: int, ktype: str = "K_DAY") -> pd.DataFrame: ...
    def quote(self, symbol: str) -> QuoteSnapshot: ...
    def financial(self, symbol: str, field: str) -> float | None: ...
        # → financial_pit.get_financial_value(symbol, field, as_of=self.now)
    def universe(self) -> list[str]: ...
        # → survivorship_bias.get_universe_for_backtest(check_date=self.now)

    # ---- 账户面 ----
    def position(self, symbol: str) -> Position: ...
    @property
    def cash(self) -> float: ...
    @property
    def equity(self) -> float: ...

    # ---- 执行面 ----
    def order(self, intent: OrderIntent) -> str: ...       # 返回 order_id
    def cancel(self, order_id: str) -> bool: ...
    def open_orders(self) -> list[OrderUpdate]: ...

    # ---- 日志（structlog，实盘同时落 Bot 日志流） ----
    def log(self, event: str, **kw) -> None: ...
```

**接线即治理**：`financial()` 与 `universe()` 是 DQ-01/02 的**唯一**策略侧入口。策略无法拿到 `announce_date > ctx.now` 的财务数据，也无法拿到「当日尚未上市/已退市」的标的——这两类偏差从「靠 review 把关」变为「类型系统不可表达」。

## 四、双 Driver 设计

### 4.1 BacktestDriver（`drivers/backtest.py` + `sim_broker.py`）

**职责**：历史回放主循环 + 模拟撮合。取代 `EventDrivenBacktestEngine` 的引擎职责（撮合逻辑抽取复用，不重写）。

```
run(strategy_cls, params, config: BacktestConfig) -> BacktestResult
  1. 构造 RunManifest（code_hash = sha256(源码)，seed 固定注入 numpy）
  2. 数据装载：KlineWarehouse.get_history()（唯一路径，废弃遗留 parquet_db 分支）
     └─ DQ-03 就绪后：按 data_snapshot_id 装载指定快照
  3. SimClock 逐 bar 推进：
     for bar in bars:
         clock.set(bar.dt)
         sim_broker.match_open_orders(bar)     # 先撮合挂单/止损（用当前 bar 高低价）
         strategy.on_bar(ctx, bar)             # 再驱动策略
         sim_broker.dispatch_fills(strategy)   # 成交回报 → on_order_update
         equity_curve.append(...)
  4. 输出 BacktestResult{metrics, equity_curve, trades, manifest}
```

关键决策：

- **撮合语义沿用现有实现**：`sim_broker.py` 从 `EventDrivenBacktestEngine` 平移限价撮合、止损、滑点、摩擦成本逻辑，保持回测结果与现网可对账；仅将「引擎持有策略」倒置为「Driver 驱动策略」。
- **history() 的实现**：Driver 预加载全量 DataFrame，`ctx.history(n)` 返回 `df.iloc[:cursor+1].tail(n)` 视图——O(1) 切片，无逐 bar 拷贝，满足矢量化性能约束（禁 iterrows）。
- **同步执行**：回测主循环是纯 CPU 任务，整体包在 `asyncio.to_thread` 内由路由调用（沿用现有模式）。
- **动态代码安全**：沿用现有 AST 沙箱（`SandboxSecurityVisitor`）审查用户源码，审查对象从旧基类切换为新 `Strategy` 契约。

### 4.2 VectorBT 快路径（`drivers/vector.py`，QUANT-01 合并设计）

**定位**：不是第二种语义，而是**同一策略的加速执行计划**。

```
适用条件：strategy.signals(df, params) 返回非 None
执行：signals → vbt.Portfolio.from_signals(...)（费率/滑点参数与 SimBroker 同源配置）
消费方：BT-03 Walk-Forward（滚动 N 窗）、BT-05 网格搜索（N 组参数）、
        BT-06 过拟合检测、BT-04 蒙特卡洛——批量场景必须走快路径，
        单次可视化回测默认走事件驱动（保留 debug 逐 bar 日志能力）
```

**同构校验器（`verify.py`）——本设计的质量核心**：

```
verify_isomorphism(strategy_cls, params, df) -> IsomorphismReport
  1. 同一份数据分别跑 事件驱动 与 VectorBT 路径
  2. 断言：交易列表（时间/方向/数量）逐笔一致；
     总收益/最大回撤相对误差 < 1e-6（浮点容差）
  3. 不一致 → 报告首个分歧 bar + 双方状态快照
```

- 实现了 `signals()` 的策略，CI 契约测试**强制**跑同构校验（Golden 数据集：固定 3 组历史行情 Parquet fixture，禁真实网络请求）。
- 校验失败 = 策略的矢量化声明作废，寻优结果不可信——这是防止「网格搜索选出的参数在事件语义下失效」的唯一防线，也是 LEAN 不具备而我们必须做的（因为我们允许双轨）。

### 4.3 LiveDriver（`drivers/live.py`）

**职责**：接管 `BotRuntimeManager._run_bot_loop` 的策略执行段，Bot 管理外壳（注册/暂停/恢复/日志流）不动。

```
生命周期（每个 Bot 一个 asyncio.Task，沿用现状）：
  1. on_init：声明订阅 → LiveDriver 向行情总线注册兴趣
  2. 行情消费：订阅 Redis Pub/Sub `quant:quotes:stream`（Protobuf 反序列化）
     └─ 降级链路：总线 30s 无消息 → 回退 futu_service.get_quote() 轮询（现状行为），
        QuoteSnapshot.stale = True
  3. K线聚合：tick 流按策略声明的 ktype 聚合成 bar（分钟界闭合才触发 on_bar），
     聚合器状态驻留内存 + Redis 断点续聚（Bot 重启不丢当前 bar）
  4. 策略调用：await asyncio.to_thread(strategy.on_bar, ctx, bar)
     └─ 修复现状：策略同步执行卡事件循环的问题
  5. ctx.history()：优先 KlineCacheEngine（L1 Redis / L2 Parquet），冷路径 Futu
  6. ctx.position()/cash：读 OMS Redis（quant:oms:positions:{market}）+ 账户缓存
  7. ctx.order() → ExecutionGateway（见 §五）
  8. on_order_update：订阅 `oms:orders:update` Pub/Sub，过滤本策略 tag 后回调
```

**paper 模式（PT-01 前置）**：LiveDriver + SimBroker 的组合——真实行情流驱动、模拟撮合记账。这使 PT-01「常驻纸面组合」变成纯配置项（`mode="paper"`），不需要新引擎。

### 4.4 时钟抽象（`clock.py`）

```python
class Clock(Protocol):
    def now(self) -> datetime: ...

SimClock   # 回测：由 Driver 逐 bar set()；策略侧只读
WallClock  # 实盘/纸面：datetime.now(UTC)
```

策略/Context 内**禁止**出现 `datetime.now()` 直调（ruff 自定义规则拦截），一切时间经 Clock——这是同构的根基。

## 五、ExecutionGateway 与安全锁（`gateway.py` + `adapters/oms_adapter.py`）

```
OrderIntent
   │
   ▼
ExecutionGateway.submit(intent, mode)
   ├─ mode ∈ {backtest, paper} ──→ SimBroker.submit()（内存撮合，无副作用）
   └─ mode == live:
        ①检查 REAL_TRADE_EXECUTE（env，全局硬锁）
        ②检查 quant:oms:trading_mode == LIVE（Redis，运行时软锁）
        ③检查 oms:kill_switch 未触发
        任一不通过 → 降级为 paper 语义 + logger.warning("[SANDBOX] ...")
        全部通过 → OmsExecutionAdapter:
            oms_service.create_order(...)          # 先落库（PENDING）
            await asyncio.to_thread(futu.place_order, ...)  # Futu 同步 SDK 隔离
            状态回写 + 发布 oms:orders:update
```

- **状态枚举收口**：`OrderStatus` 以 `schemas/domain.py` 为准，新增 `PARTIALLY_FILLED` 别名迁移（DB 存量值映射），OMS 写库改用枚举值——不一致问题在适配层一次性修正。
- **幂等**：`OrderIntent.tag` + run_id 生成 client_order_id，重复提交（Bot 重启补发）由 Gateway 去重。
- 复用现有 Kill Switch / 算法单（algo_engine）不在 V1 范围；Gateway 预留 `algo: str | None` 字段即可（YAGNI）。

## 六、迁移策略（不停机、分阶段）

| 阶段 | 内容 | 兼容措施 |
|:---|:---|:---|
| M1 | `backend/engine/` 契约层落地，与现网零耦合 | 纯新增，无风险 |
| M2 | BacktestDriver 上线，`/strategy/run-sandbox` 增加 `engine=v2` 参数双跑 | `adapters/legacy.py` 将旧 `_generate_signals` 策略包装为新契约；新旧结果对账一周 |
| M3 | VectorBT 快路径 + 同构校验器，寻优/WF/蒙特卡洛路由切到快路径 | 校验器纳入 CI |
| M4 | ExecutionGateway + LiveDriver，`deploy-to-oms` 生成新契约策略 | 存量 3 个 live 策略手工改写（本来就无 on_tick，无行为损失）；`bot_runtime` 保留外壳，策略执行段委托 LiveDriver |
| M5 | 旧路径下线：`EventDrivenBacktestEngine` 对外入口移除、`backend/backtest/` 标记 deprecated | 归档前跑全量回归 |

回滚策略：M2~M4 每阶段均由环境变量开关（`ENGINE_V2_ENABLED`）控制，异常即回切旧路径。

## 八、风险与开放问题

| # | 风险/开放问题 | 处置 |
|:---|:---|:---|
| 1 | 撮合逻辑平移可能引入行为漂移，历史回测结果变化引发信任问题 | M2 双跑对账一周；Golden fixture 锁定旧引擎输出为基线 |
| 2 | 行情总线 Protobuf 消息覆盖标的不全（仅 market_engine 订阅列表） | LiveDriver 降级轮询兜底 + Bot 声明订阅时向 market_engine 注册标的（配额计数器复用） |
| 3 | `signals()` 与 `on_bar` 语义等价对策略作者是负担 | 校验器给出首个分歧 bar 的双方状态快照，定位成本可控；不实现 signals() 只是失去批量寻优加速，功能不受损 |
| 4 | tick→bar 聚合在低流动性标的上 bar 稀疏，60s 轮询模式下分钟 bar 可能缺失 | V1 实盘仅承诺 K_DAY / K_60M 粒度；分钟级依赖行情总线推送密度，文档明示 |
| 5 | 动态 exec 策略进实盘的安全边界（现状 importlib 无沙箱） | V1 决策：实盘策略部署仍走文件落盘 + AST 静态审查（与回测同一套 Visitor），不引入运行时沙箱；进程级隔离列为 V2 观察项 |
| 6 | PIT store 当前为内存单例，实盘长驻进程数据从哪来 | V1 `financial()` 在 live 模式允许返回 None（数据未装载），策略需判空；PIT 持久化依赖 DQ-03 排期 |
