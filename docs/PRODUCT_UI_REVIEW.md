## 二、信息架构重设计（最高优先级）

### 2.1 当前架构的核心问题

```
当前：分页跳转模式（Page-per-Feature）
用户查行情 → [/quotes 页面]
用户看新闻 → [跳转至 /data-center 页面，行情图消失]
用户用 AI → [跳转至 /copilot 页面，一切上下文丢失]
用户看风控 → [跳转至 /risk 页面，选股结果消失]

问题：每次页面跳转都是一次"上下文清空"，与专业终端的"上下文持续"理念背道而驰。
```

- [ ] **情感热度列**：显示过去 7 天 AI 新闻情感得分（红绿色块），无需离开页面
- [ ] **结果内联迷你图**：每行标的的小型 Sparkline 走势图（Lightweight-Charts 极小 size）

## 五、视觉设计专项优化

### 5.1 数据密度分级

量化终端数据密度高，必须分级处理：

| 级别 | 场景 | 字号 | 行高 | 颜色对比度 |
|:---|:---|:---|:---|:---|
| **极密** | 盘口挂单、Tick 流 | 11px mono | 20px | 高（白/绿/红） |
| **密** | AG Grid 数据行 | 12px | 28px | 标准 |
| **标准** | 功能卡片、摘要 | 14px | 32px | 标准 |
| **稀疏** | 告警标题、状态 | 16px | 40px | 可低对比度 |

### 5.2 Loading State 规范

```
数据加载中：骨架屏（Skeleton），色调 bg-slate-800/50，配合 animate-pulse
数据为空时：空状态插图 + 操作引导（"还没有持仓，去选股器找标的？"）
数据失败时：错误卡片 + 重试按钮 + 错误 Code（供用户截图反馈）
数据过期时：STALE Badge + 最后更新时间 + 重连倒计时
```

### 5.3 颜色语义扩展

在原有红绿基础上增加细粒度语义：

```
极强看涨（+3%以上）  → text-emerald-300 font-bold
强看涨（+1~3%）      → text-emerald-400
弱看涨（0~1%）       → text-emerald-600
平盘（±0.1%以内）    → text-slate-400
弱看跌（0~-1%）      → text-red-600
强看跌（-1~-3%）     → text-red-400
极强看跌（-3%以下）  → text-red-300 font-bold

风险提示色（⚠️）    → text-amber-400 bg-amber-500/10
信息提示色（ℹ️）    → text-blue-400 bg-blue-500/10
成功确认色（✅）    → text-emerald-400 bg-emerald-500/10
```
