### 1.2 必须修复的问题（高优先级）

| 问题 | 影响 | 修复方案 |
|:---|:---|:---|
| **双布局系统并存** | AI 生成代码时随机选用，导致代码污染 | 废弃 `DashboardLayout`（Next.js 路由），统一使用 `TradingDashboard`（模块切换） |
| **TickerTape 轮询** | 数据最多滞后 60s | 迁移至 WebSocket `/ws/v1/quotes` 订阅 |
| **WS 断连无 STALE 遮罩** | 用户看到过期数据无感知 | 断连时在所有实时数据区域叠加 STALE 遮罩 |

| **无 Cmd+K 绑定** | OmniSearch 组件已有但无快捷键入口 | 在 `TradingDashboard` 级别绑定全局键盘事件 |
| **无沙箱/实盘模式标识** | 用户可能误操作真实账户 | 顶栏显示醒目的模式横幅 |
| **移动端无响应式方案** | 手机端布局崩溃 | 定义移动端降级视图（只读监控） |

## 二、全局布局体系（Layout Architecture）

### 2.1 布局决策：模块切换 vs 页面路由

**确定选用：`TradingDashboard` 模块切换模式**

原因：量化终端是长期挂机的重型客户端，用户在模块间频繁切换时需要**保留各模块的内部状态**（K 线缩放级别、选股条件、策略代码编辑进度）。页面路由会在每次跳转时卸载组件、清空状态，对用户体验破坏极大。

```
TradingDashboard（根容器）
  ├── Navbar（全局顶栏，固定 48px）
  ├── MainArea（flex-row，overflow-hidden）
  │   ├── Sidebar（模块导航，56px 折叠 / 220px 展开）
  │   └── ModuleContainer（flex-1，overflow-auto）
  │       └── [当前激活模块] — 其他模块 hidden 但不 unmount
  └── StatusBar（全局底部状态栏，固定 28px）
```

### 2.2 模块保活策略（Keep-Alive Pattern）

```tsx
// trading-dashboard.tsx — 所有模块同时挂载，通过 display 切换可见性
// 避免切换模块时 K线缩放状态、策略代码等丢失

{Object.entries(moduleComponents).map(([id, Component]) => (
  <div
    key={id}
    className={cn(
      "absolute inset-0 overflow-auto transition-opacity duration-150",
      activeModule === id
        ? "opacity-100 pointer-events-auto z-10"
        : "opacity-0 pointer-events-none z-0"
    )}
    aria-hidden={activeModule !== id}
  >
    <ErrorBoundary moduleId={id}>
      <Component />
    </ErrorBoundary>
  </div>
))}
```

### 4.5 模块切换时的 UX 守则

```
切换模块时的行为规范：
1. 已激活模块不卸载（keep-alive），内部状态保留
2. 不重置目标模块的滚动位置（用户上次在哪，回来还在哪）
3. 模块切换过渡：opacity 150ms，不用 translateX（避免视觉晕眩）
4. 侧边栏的激活状态立即更新（无延迟）
5. 浏览器 URL 不变（不使用路由，避免浏览器 back/forward 混乱）
```

### 4.6 WebSocket 断连处理流程

```
WebSocket 断连事件触发
  ↓
1. 立即：底部状态栏 WS 指示灯变红
   立即：所有实时数据区域显示 STALE 遮罩（半透明 + 警告文字）
   立即：TickerTape 停止滚动，最后数据加上 "⚠" 标记
   ↓
2. 1s 后：自动重连（指数退避：1s / 2s / 4s / 8s）
   重连期间：底部状态栏显示 "正在重连 (1/5)..."
   ↓
3. 重连成功：
   STALE 遮罩淡出（300ms transition）
   底部状态栏恢复绿色
   TickerTape 重新滚动
   ↓
4. 5次重连失败：
   显示全局 Toast 通知："行情连接中断，请检查网络"
   底部状态栏持续红色
   建议操作按钮："重新加载"
```

## 六、零 GC 数据管道（Zero-GC Pipeline）

### 6.1 高频数据通路（Tick → 图表）

```
WebSocket 帧到达 (每秒 100-500 次)
  ↓
useQuoteStream Hook（不触发任何 React 状态更新）
  ↓
Float64Array 环形缓冲区（useRef，堆内存固定，无 GC）
  ↓
requestAnimationFrame 节流（最高 60fps，跳过中间帧）
  ↓
chart.update(latestTick)  ← 直接调用图表实例方法
  ↓
Canvas 重绘（绕过 React VDOM，零 Diff 开销）
```

**严禁路径**：
```
WebSocket 帧 → setState → React Diff → DOM Patch  ← 这会 GC 爆炸
```

### 6.2 渲染引擎选用（铁律，不允许混用）

| 场景 | 必须用 | 严禁用 |
|:---|:---|:---|
| K线主图、分时图 | `Lightweight-Charts` | ECharts、Recharts |
| Level 2 盘口挂单墙 | `PixiJS v8` (WebGL) | DOM Table 渲染 |
| 5000+ 行数据列表 | `AG Grid` 虚拟滚动 | 原生 `<table>`、普通列表 |
| AI 生成图表、归因图 | `Apache ECharts` | PixiJS |
| 实时数字（价格跳动） | `useRef` + 直接 DOM 写入 | `useState`（触发重渲染） |

### 6.3 Zustand Store 高频安全规范

```tsx
// ✅ 正确：只在 Store 存低频业务状态
interface MarketStore {
  subscribedSymbols: Set<string>    // 订阅列表（低频）
  activeSymbol: string              // 当前查看的标的（低频）
  // ❌ 错误示例（不能放 Store 的）：
  // tickData: Tick[]               // 高频！会导致全局重渲染
  // klineData: OHLCV[]             // 高频！
}

// ✅ 高频数据放 ref，在组件内局部管理
const tickBufferRef = useRef<Float64Array>(new Float64Array(1000))
const klineRef = useRef<KlineBar[]>([])
const chartInstanceRef = useRef<IChartApi | null>(null)
```

## 七、错误处理与边界防护

### 7.1 三层 Error Boundary

```tsx
// 第一层：页面级 → 整个模块崩溃时显示模块错误卡片
// components/feedback/ModuleErrorBoundary.tsx
class ModuleErrorBoundary extends React.Component {
  state = { hasError: false, error: null }
  static getDerivedStateFromError(error) { return { hasError: true, error } }
  render() {
    if (this.state.hasError) return <ModuleErrorCard error={this.state.error} onReset={...} />
    return this.props.children
  }
}

// 第二层：面板级 → 单个面板崩溃时显示面板错误提示
<PanelErrorBoundary fallback={<div className="text-red-400 text-xs p-4">面板加载失败，请刷新</div>}>
  <OrderBookPanel />
</PanelErrorBoundary>

// 第三层：图表级 → 单个图表崩溃时显示"图表不可用"
<ChartErrorBoundary symbol={symbol}>
  <LightweightChartCanvas />
</ChartErrorBoundary>
```

### 7.2 网络请求统一错误处理

```tsx
// lib/api-client.ts — 拦截器统一处理
apiClient.interceptors.response.use(
  (res) => res,
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期 → 尝试静默刷新，失败则跳转登录
      await refreshToken()
    }
    if (error.response?.status === 429) {
      // 限速 → 显示 Toast 提示，不崩溃
      toast.warning("操作过于频繁，请稍后再试")
    }
    if (error.response?.status >= 500) {
      // 服务端错误 → 显示错误 Toast + 错误码
      toast.error(`服务异常 (${error.response?.data?.error_code ?? 'UNKNOWN'})`)
    }
    return Promise.reject(error)
  }
)
```

```tsx
// 生产环境监控渲染帧率，低于 45fps 时告警
function useFrameRateMonitor() {
  useEffect(() => {
    let frames = 0, lastTs = performance.now()
    const monitor = (ts: number) => {
      frames++
      if (ts - lastTs >= 2000) {
        const fps = Math.round(frames * 1000 / (ts - lastTs))
        if (fps < 45) logger.warn('low_frame_rate', { fps, module: activeModule })
        frames = 0; lastTs = ts
      }
      rafId = requestAnimationFrame(monitor)
    }
    let rafId = requestAnimationFrame(monitor)
    return () => cancelAnimationFrame(rafId)
  }, [])
}
```
