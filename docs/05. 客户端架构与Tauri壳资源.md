# 客户端架构：Flutter 薄客户端（Android / iOS / HarmonyOS）(V4.1)

> **文档定位**：移动端实施前架构 SSOT——整洁架构分层、模块边界、统一接口、端侧极简原则、现代 UI / Figma 指引。  
> **最后更新**：2026-07-13 | **版本**：V4.1  
> **框架终审**：ADR-002 / ADR-006 — **Flutter 三端**；否决 Tauri Mobile / 原生四端（文件名保留历史「Tauri」字样，内容以 Flutter 为准）  
> **关联**：`docs/01` 产品 IA · `docs/04` Web 前端 · `docs/02` SSOT · `docs/18` 推送 · `docs/TODO.md` CLI-*  
> **原则一句话**：**业务与算力在后端；客户端只做鉴权、订阅、展示、确认与推送落地。**

## 〇、实施前 Review 结论（V4.0）

### 0.1 V3.0 问题清单

| 问题 | 风险 | V4.0 纠正 |
|:---|:---|:---|

| 端侧重造图表/选股/策略 IDE | 单兵不可维护 | **薄客户端**：不做策略 IDE / 完整 Screener / 回测工坊 |
| 自研 CustomPainter K 线作为 P0 | 工期爆炸 | P0 用后端摘要 + 轻量图；重度 K 线延后或另立 ADR |
| Dio + 平行 HTTP 叙事 | 与 Web「单一 Gateway」不一致 | **唯一 Gateway 接口层**（Ports） |
| 视觉规范有 Token，无 Figma 流程 | 设计无法协作落地 | 新增 §八 Figma Design System 指引 |
| Phase 4 备选过多 | 稀释主线 | 主线仅随身监控 + 告警 + 简化交易 |

### 0.2 不可推翻决策

1. **框架**：Flutter 3.22+ / Dart 3.4+（含 HarmonyOS Fork）  
2. **定位**：随身监控 / 告警确认 / 简化下单；**重度投研留在 Web**（`docs/01 §1.3`）  
3. **数据 SSOT**：一律经后端 Gateway（禁止端上直连 Futu / YFinance / Redis）  
4. **依赖**：只允许 **外层 → 内层**；Domain 零 Flutter / 零网络

## 一、整洁架构（Clean Architecture）

### 1.1 分层与依赖方向

```
┌─────────────────────────────────────────────────────────────┐
│  Presentation（UI）                                         │
│  pages / widgets / theme / go_router shells                 │
│  只依赖 Application；禁止 import infrastructure 实现类        │
└───────────────────────────┬─────────────────────────────────┘
                            │ ↓
┌───────────────────────────▼─────────────────────────────────┐
│  Application（Use Cases / Controllers）                     │
│  Riverpod Notifiers · 用例编排 · 订阅生命周期                 │
│  只依赖 Domain Ports；禁止 import Dio/Isar/平台通道实现        │
└───────────────────────────┬─────────────────────────────────┘
                            │ ↓
┌───────────────────────────▼─────────────────────────────────┐
│  Domain（Entities + Ports）                                 │
│  纯 Dart：实体、值对象、Repository/Gateway 抽象接口           │
│  禁止：flutter/*、dio、isar、firebase、platform channels     │
└───────────────────────────┬─────────────────────────────────┘
                            │ ↑ 实现注入
┌───────────────────────────▼─────────────────────────────────┐
│  Infrastructure（Adapters）                                 │
│  GatewayHttp · WsGateway · SecureStore · Push · APM         │
│  实现 Domain Ports；可依赖 Flutter / 平台 SDK                 │
└─────────────────────────────────────────────────────────────┘
```

**依赖铁律**：

| 从 \\ 到 | Presentation | Application | Domain | Infrastructure |
|:---|:---:|:---:|:---:|:---:|
| Presentation | — | ✅ | ⚠️ 仅实体/枚举 | ❌ |
| Application | ❌ | — | ✅ | ❌（经 Port） |
| Domain | ❌ | ❌ | — | ❌ |
| Infrastructure | ❌ | ❌ | ✅ 实现 Port | — |

> DI：在 `main.dart` / `injection.dart` 用 Riverpod `Provider` 把 Infrastructure 实现绑到 Domain Port。Application / Presentation **只拿 Port 类型**。

### 1.2 统一接口层（Gateway Ports）

与 Web `lib/api-client.ts` 对齐——**端上只有这一条出站通道契约**：

```dart
// domain/ports/quant_gateway.dart  （示意）

abstract interface class QuantRestGateway {
  Future<ApiResult<T>> get<T>(String path, {Map<String, dynamic>? query});
  Future<ApiResult<T>> post<T>(String path, {Object? body});
}

abstract interface class MarketStreamGateway {
  Stream<QuoteTick> subscribeQuotes(Set<String> symbols);
  Stream<AlertPush> subscribeAlerts(); // 对齐 docs/18 WS 频道
  Future<void> pause();  // App 进后台
  Future<void> resume();
}

abstract interface class AuthTokenStore {
  Future<void> saveTokens({required String access, required String refresh});
  Future<String?> readAccessToken();
  Future<void> clear();
}

abstract interface class AppTelemetry {
  void recordWsLatency(Duration d);
  Future<void> flushHeartbeat(); // → POST /api/v1/client/heartbeat
}
```

**禁止**：

- Feature 内 `Dio()` / `WebSocket.connect` 直连  
- 平行第二套 HTTP 客户端  
- Hermes Tool / 外部行情 URL 从端上直打（必须走后端）

### 1.3 薄客户端边界（Minimize Client Logic）

| 能力 | Web（docs/04） | Flutter 客户端 | 理由 |
|:---|:---|:---|:---|
| 行情盯盘 / 自选 | ✅ 完整 | ✅ **精简**（列表 + 简 K / 跳转 Web） | 随身监控 |
| 选股 Screener | ✅ | ❌ 不做 NLP 构建器；最多「结果只读」深链 | 算力与交互在 Web |
| 策略实验室 / Monaco | ✅ | ❌ | 键盘 + 大屏 |
| 回测工坊 / Tear Sheet | ✅ | ❌；推送「回测完成」+ 深链 Web | 报告密度过高 |
| OMS 持仓 / 撤单 / Kill | ✅ | ✅ **简化** + LIVE 生物识别 | 关键路径 |
| 风控全景 | ✅ | ⚠️ KPI 卡片只读 | 图表留 Web |
| Copilot | ✅ 全局抽屉 | ✅ 精简 SSE 对话 | 复用后端 SSE |
| 告警中心 | ✅ | ✅ **P0 场景**（推送 + 确认 + 跳转） | docs/18 |
| 纸面组合 | ✅ | ⚠️ NAV / TE 摘要只读 | docs/17 |

**端上允许的计算**：格式化、本地缓存命中判断、UI 状态机。  
**端上禁止的计算**：因子、回测、选股 DSL、技术指标重算（若需要由后端 API / Worker 提供）。

## 三、技术栈（Frozen · 与薄客户端匹配）

```
Flutter SDK        : 3.22+（Dart 3.4+）
鸿蒙               : 华为 flutter-harmonyos Fork（按需 Phase 2）
状态               : Riverpod 2.x（Notifier / AsyncNotifier）
路由               : go_router（深链 = 与推送 ui_hint 对齐 docs/18）
HTTP / WS / SSE    : 仅经 Infrastructure Gateway 实现（底层可用 http/dio/web_socket_channel，对外不可见）
安全存储           : flutter_secure_storage
推送               : APNs / FCM / HMS（Infrastructure/push）
本地缓存           : 可选 Isar/Hive — 仅缓存 Gateway 已返回的 DTO，不做衍生计算
图表               : P0 轻量（sparkline / 官方 charts 或简单 CustomPainter）
                     P2+ 重度 K 线另立 ADR；禁止默认 WebView 嵌 ECharts（除非 ADR 例外）
APM                : AppTelemetry Port → POST /api/v1/client/heartbeat
```

**严禁**：

- ❌ Feature 内 `setState` 灌高频 Tick（用 Stream + 局部 Repaint / 监听器）  
- ❌ `SharedPreferences` 存 Token  
- ❌ 主 Isolate 解析超大 JSON（`compute` / Isolate）  
- ❌ 引入第二套状态库（Bloc + Riverpod 混用）  
- ❌ 端上实现选股 DSL / 回测引擎

## 六、交互与安全（精简保留）

### 6.1 WS 生命周期

后台 `paused` → `MarketStreamGateway.pause()`；`resumed` → `resume()`；断连指数退避 + STALE。

### 6.2 鉴权

- Token 仅 `AuthTokenStore`（Keychain / Keystore / OHOS）—— **CLI-04** 已落地：`SecureAuthTokenStore` + `FlutterSecureKvStore`  
- Dio 请求经 `AuthBearerInterceptor` 注入 Bearer；登录页 `/login` + guest 路由守卫  
- 注：后端 refresh 目前偏 httpOnly Cookie（Web）；移动端仅在响应 JSON 含 `refresh_token` 时落盘，Cookie-only refresh 待后续对齐  
- LIVE 下单 / Kill / 查看密钥：生物识别  
- Kill：输入确认短语 + 生物识别  

### 6.3 推送落地（docs/18）

- **CLI-05** 已落地：`PushNotificationPort` → FCM / APNs / HMS（CLI-06 真 Channel）+ `MemoryPushAdapter`（测试）  
- 消费 `ui_hint`（如 `{route, symbol}`）→ `resolveAlertNavigation` → `go_router` 深链（`/market`+symbol → `/quotes/:symbol`）  
- P0：`AlertOverlayHost` 全屏确认；P1/P2：SnackBar Toast；P3：Tab 🔔 角标  
- Token 软注册 `POST /api/v1/client/push-token`（后端未就绪时失败可忽略）  
- 冷启动：`consumeInitialMessage()` 入队并深链  
- **CLI-06**：`platform/harmonyos/` MethodChannel（`hms_push` / `hms_auth`）+ `loginWithHms` → `/api/v1/auth/hms`；`--dart-define=HARMONYOS=true`；`ohos/README` 契约

## 八、Figma 设计工具指引（Design System）

> 目标：设计师在 Figma 产出可被工程 1:1 消费的 Token 与组件，避免「效果图无法落地」。

### 8.1 文件与页面结构

```
QuantEdge Mobile DS（Figma 文件）
├── 0 · Cover / Changelog
├── 1 · Foundations
│     Colors · Typography · Spacing · Radius · Elevation · Icons
├── 2 · Components
│     AppBar · TabBar · ModeBanner · PriceText · ChangeBadge
│     StaleOverlay · ListRow · OrderSheet · AlertCard · KillFab
├── 3 · Patterns
│     Portfolio Home · Quotes List · Alert Inbox · Copilot Thread · Settings
├── 4 · Flows
│     Login · Push → Deep Link · LIVE Order Confirm · Kill Switch
└── 5 · Specs
      Redlines · Token JSON 导出说明
```

### 8.2 Figma 操作规范

1. **Variables（变量）**  
   - 建 Collection：`Color/Semantic`、`Color/Raw`、`Space`、`Type`  
   - Semantic 映射 §7.1；禁止组件直接绑 Raw 色  
2. **Modes**  
   - 至少 `Dark`（默认生产）；`Light` 可选  
3. **Auto Layout**  
   - 所有组件必须 Auto Layout；间距只用 `Space/*` 变量（4 基准栅格）  
4. **Components / Variants**  
   - `PriceText`：`direction=up|down|flat` · `size=sm|md|lg`  
   - `ModeBanner`：`mode=sandbox|paper|live`  
   - `ListRow`：`state=default|pressed|stale`  
5. **Dev Mode**  
   - 标注：触控热区 ≥ 44×44；安全区；动态岛/挖孔避让  
6. **导出给工程**  
   - Variables → 导出 JSON → 生成 `presentation/theme/color_tokens.dart`（可用 Style Dictionary / 手写同步表）  
   - 图标：统一 SVG → `flutter_svg`；禁止位图图标散落  

### 8.3 设计 ↔ 代码对照表

| Figma | Flutter |
|:---|:---|
| Variables / Color | `AppColors` / `ColorScheme` extension |
| Text Styles | `AppTextStyles` + `fontFeatures: tabularFigures` |
| Component | `presentation/widgets/*` |
| Prototype Flow | `go_router` routes + push `ui_hint` |
| Breakpoint 600 | `MobileShell` / `TabletShell` |

**CLI-ARCH-02 已落地同步表**：

| 产物 | 路径 |
|:---|:---|
| 机器可读 Variables 表 | `client/flutter_app/design/figma_variables_sync.json` |
| 人读对照 | `client/flutter_app/design/FIGMA_TOKEN_SYNC.md` |
| Dart Token | `lib/presentation/theme/color_tokens.dart`（`AppColors` / `AppSpace` / `AppRadius`） |
| 回归测试 | `test/cli_arch02_figma_token_sync_test.dart` |

改 Figma Semantic 色后：先改 JSON → 再改 `color_tokens.dart` → 跑 ARCH-02 测试。
