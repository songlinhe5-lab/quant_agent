## 子系统一：前端 Web Terminal

**详细架构**：`docs/subsystems/frontend/architecture.md`

| 属性 | 描述 |
|:---|:---|
| **技术栈** | React 18 + Vite SPA + TypeScript + Tailwind CSS v4 |
| **路由** | React Router v6（客户端路由） |
| **状态管理** | Zustand（低频业务状态） + useRef（高频 Tick） |
| **渲染引擎** | Lightweight-Charts / PixiJS / AG Grid / ECharts（分级） |
| **通信协议** | WebSocket（行情）+ SSE（AI 流）+ fetch（REST） |
| **部署方式** | Nginx 静态服务 / Cloudflare Pages |

**入口文件**：`frontend/src/main.tsx`  
**路由配置**：`frontend/src/router/index.tsx`  
**全局状态**：`frontend/src/stores/`

**核心数据流**：
```
WebSocket 行情 → useRef(Float64Array) → 图表 .update() → Canvas 渲染
SSE Agent 流  → ChatStore(Zustand) → React 重渲染  → 打字机效果
REST API      → fetch → 组件 state → React 渲染
```

## 子系统二：后端 API 网关与执行引擎

**详细架构**：`docs/subsystems/backend/architecture.md`

| 属性 | 描述 |
|:---|:---|
| **技术栈** | Python 3.11 + FastAPI + asyncio + Pydantic v2 |
| **IPC** | ZeroMQ（进程间 microsecond 级） |
| **消息总线** | Redis Pub/Sub + Streams |
| **持久化** | SQLAlchemy 2.0 + asyncpg → PostgreSQL |
| **分析存储** | DuckDB + Parquet（本地数据湖） |

**目录职责速查**：

| 目录 | 职责 | 不允许的行为 |
|:---|:---|:---|
| `backend/routers/` | HTTP/WS 路由，参数校验 | 写业务逻辑 |
| `backend/services/` | 业务逻辑，编排调用 | 直接写 SQL / 调外部 API |
| `backend/workers/` | 长驻后台任务（行情 / OMS） | 处理 HTTP 请求 |
| `backend/core/` | 基础设施（DB / Redis / 日志）| 写业务逻辑 |
| `backend/models/` | SQLAlchemy ORM 模型 | 写查询逻辑 |
| `backend/schemas/` | Pydantic 请求/响应 Schema | 写业务逻辑 |

**API 端点速查**：见 `docs/backend.md` §6

## 子系统三：Hermes Agent 推理引擎

**详细架构**：`docs/subsystems/agent/architecture.md`

| 属性 | 描述 |
|:---|:---|
| **框架** | 自研 ReAct 引擎 |
| **工具数量** | 14 个标准工具（见 AGENTS.md §2） |
| **LLM** | GPT-4o（主）/ Ollama Qwen2.5（降级） |
| **推理循环** | Plan → Tool → Verify → Output |
| **输出流** | SSE（Server-Sent Events）逐 token 推送 |
| **RAG 存储** | pgvector（PostgreSQL 向量扩展） |

**Tool 调用约定**：
- Tool 函数统一接受 `str` 参数，返回 `dict`
- 每个 Tool 必须有对应 `tests/tools/test_xxx_tool.py`
- Tool 内部不调用其他 Tool（防循环依赖）
- Tool 失败必须抛出 `ToolExecutionError`（含错误码和描述）

**新增 Tool 流程**：
1. 在 `hermes_agent/tools/` 下创建 `xxx_tool.py`
2. 在 `hermes_agent/tool_registry.py` 注册
3. 在 `AGENTS.md §2` 中添加工具说明
4. 在 `tests/tools/` 下创建测试文件
5. 更新 `docs/subsystems/agent/architecture.md`

## 子系统边界规则（Boundary Rules）

> 跨子系统的数据流必须经过明确的接口，不允许"内部捷径"。

```
✅ 允许的调用方向：
  Frontend     → Backend Gateway（HTTP / WebSocket）
  Hermes Agent → Backend Gateway（内网 HTTP）
  Backend Gateway → Services → Workers → External APIs
  Workers        → Redis（数据总线）← Backend Gateway

❌ 禁止的调用方向：
  Frontend     → External APIs 直连（Futu / YFinance 等）
  Hermes Tool  → External APIs 直连
  Mobile App   → Redis / PostgreSQL 直连
  Router       → External APIs 直连（必须经 Service）
```
