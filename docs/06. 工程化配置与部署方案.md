> **文档定位**：量化交易终端工程化与**四节点**生产部署规范（数据源分 VPS + Tailscale 虚拟网 + YFinance 多 IP 抗限流）。  
> **最后更新**：2026-07-13 | **版本**：V9.0  
> **拓扑**：美国主 VPS ×1 + 美国 YFinance 辅助 VPS ×2 + 中国 AKShare 辅助 VPS ×1  
> **关联**：`docs/14` 数据源框架 · `docs/18` 推送 · `docs/TODO.md` DIST-* · `AGENTS.md` §9/§10  
> **原则**：**API/DB/OMS 集中主节点；数据源按连通性拆到专用节点；节点间只走 Tailscale，不暴露数据端口到公网。**

## 〇、部署 Review 结论（相对 V8.0）

| V8.0 问题 | 风险 | V9.0 纠正 |
|:---|:---|:---|
| 仅「一主 + 一 YF 备」 | 单备用仍易同出口限流；容量不足 | **2 个独立公网 IP 的 US YF 子服务** + 主节点可不跑 YF 直连 |
| 文档与 AGENTS 节点角色摇摆（北京主/加州主） | 部署人看错 | **冻结：US-MASTER = API SSOT**；CN = AKShare only |
| Compose / 代理 URL 混用公网 | 安全与跨境抖动 | **一律 Tailscale hostname / 100.x** |
| 国内节点也跑 YFinance | 无意义且易污染路由 | CN 节点 **禁止** YF；只 AKShare→Redis |
| 主节点本地 YF 与远程混跑未写清策略 | 限流连带打挂 API | 主节点 `YF_ROUTER_ENABLED=true`，**默认不直连 Yahoo**（或极低配额本地兜底） |

## 一、全局部署拓扑（四节点）

### 1.1 逻辑拓扑

```
                         Cloudflare Pages（前端静态）
                                    │ HTTPS → 仅 US-MASTER:8000
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│                     Tailscale Tailnet（虚拟内网）                      │
│                                                                      │
│  ┌────────────────────────────┐    ┌─────────────────────────────┐  │
│  │ US-MASTER（美国主 VPS）     │    │ US-YF-A（美国辅助 #1）        │  │
│  │ API + Worker + Redis + PG  │◄──►│ data_subservice (yfinance)  │  │
│  │ Futu OpenD @127.0.0.1      │    │ 独立公网 IP → Yahoo         │  │
│  │ YFinanceRouter / Registry  │    └─────────────────────────────┘  │
│  │ Finnhub / FRED（可本地）   │    ┌─────────────────────────────┐  │
│  │ Grafana/Prometheus(可选)   │◄──►│ US-YF-B（美国辅助 #2）        │  │
│  └──────────────▲─────────────┘    │ data_subservice (yfinance)  │  │
│                 │                  │ 独立公网 IP → Yahoo         │  │
│                 │                  └─────────────────────────────┘  │
│  ┌──────────────┴─────────────┐                                      │
│  │ CN-AKSHARE（中国辅助 VPS）  │                                      │
│  │ AKShareCollector → Redis   │  （写主节点 Redis；或 HMAC 子服务）   │
│  │ 禁止跑 YFinance / Futu     │                                      │
│  └────────────────────────────┘                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**流量边界**：

| 流量 | 路径 | 禁止 |
|:---|:---|:---|
| 用户 / Cloudflare → API | 公网 → `US-MASTER:8000`（或 Tunnel） | 前端直连辅助节点 |
| 主 → YF-A / YF-B | Tailscale → `data_subservice:8000` + HMAC | 公网互打子服务 |
| CN → 主 Redis | Tailscale → `US-MASTER:6379`（仅内网 ACL） | 公网暴露 Redis |
| OpenD | 仅 US-MASTER 宿主机 `127.0.0.1:11111` | 容器外网暴露 11111 |

### 1.2 节点职责矩阵（冻结）

| 节点代号 | 角色 | 跑什么 | 数据源 | Compose / Profile |
|:---|:---|:---|:---|:---|
| **US-MASTER** | 业务 SSOT | `quant-agent` · `quant-worker` · Redis · PostgreSQL ·（可选 monitoring）· Futu OpenD | Futu 本地；Finnhub/FRED 可本地；**YF 经 Router 调辅助**；AKShare 读 Redis/远程 | `docker-compose.master.yml` 或 `COMPOSE_PROFILES=master,monitoring` |
| **US-YF-A** | YF 出口 #1 | 仅 `data_subservice`（yfinance） | Yahoo（本机公网 IP） | `docker-compose.yf-node.yml` + `NODE_ID=us-yf-a` |
| **US-YF-B** | YF 出口 #2 | 仅 `data_subservice`（yfinance） | Yahoo（另一公网 IP） | 同上 + `NODE_ID=us-yf-b` |
| **CN-AKSHARE** | 国内连通 | AKShare Collector（或 akshare 子服务） | 东财等国内源 → **写主 Redis** | `docker-compose.slave.yml` / `COMPOSE_PROFILES=slave` |

> IP / Tailscale hostname 写入各节点 `.env`，**禁止写进仓库**。文档用代号；运维手册用私有 inventory。

### 1.3 数据源落点规则（连通性优先）

| 数据源 | 推荐落点 | 原因 | 主节点访问方式 |
|:---|:---|:---|:---|
| **Futu / OpenD** | US-MASTER 宿主机 | 券商网关与 OMS 同机，延迟最低 | `internal` TCP `host.docker.internal:11111` |
| **YFinance** | US-YF-A + US-YF-B（强制多 IP） | 429 按 **出口 IP** 计；多 VPS = 多配额池 | `external` · `YFinanceRouter` 加权轮询 |
| **AKShare** | CN-AKSHARE | 海外直连东财不稳定 | Redis cache 模式 / HMAC 远程 |
| **Finnhub** | US-MASTER（或单独 US 辅） | API Key 配额，非 IP 地理敏感 | `internal` 为主 |
| **FRED** | US-MASTER | 联储 API，美国出口稳定 | `internal` |
| **OpenAI / Ollama** | US-MASTER | 与 Hermes 同区 | `internal` |

**硬规则**：

1. **同一数据源的限流敏感调用，不得集中在单一公网 IP**（YFinance 至少 2 个 US 辅）。  
2. **主节点默认不把 Yahoo 流量打满**——Router 优先辅助节点；本地 YF 仅 `weight` 最低兜底或关闭。  
3. **CN 节点禁止启用 YFinance / Futu Collector**（避免错误注册进 Registry）。  
4. **辅助节点禁止跑 Postgres / 主 API**（无状态或仅本地缓存；注册表心跳写主 Redis）。

### 1.4 服务矩阵（US-MASTER）

| 服务 | 端口 | 说明 |
|:---|:---|:---|
| postgres | 5432 | OLTP + pgvector；仅 Tailscale / localhost |
| redis | 6379 | 缓存 + Pub/Sub + Registry；ACL 仅 Tailnet |
| quant-agent | 8000 | FastAPI；公网或 Cloudflare |
| quant-worker | — | 采集 daemon（Futu/Finnhub 等）+ 后台任务 |
| prometheus/grafana | 9090/3001 | 仅 Tailscale 访问 |

采集器环境变量（主节点示例）：

```bash
COLLECTOR_FUTU=true
COLLECTOR_FINNHUB=true
COLLECTOR_YFINANCE=false          # 推荐关闭本地采集，改走 Router
COLLECTOR_AKSHARE=false           # AKShare 在 CN 节点
YF_ROUTER_ENABLED=true
```

### 1.5 Tailscale 虚拟网规则（强制）

> **OPS-02 已完成** (2026-07-14): ACL 策略、节点入网脚本、跨节点验证脚本均已就绪。

```bash
# 四台机器均加入同一 Tailnet
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --ssh --advertise-tags=tag:quant

# 建议 MagicDNS hostname
# us-master / us-yf-a / us-yf-b / cn-akshare
```

**自动化脚本**：

| 脚本 | 用途 |
|:---|:---|
| `scripts/deploy/setup_tailscale.sh <role>` | 节点入网 + 防火墙加固 + SSH 配置 |
| `scripts/deploy/verify_tailscale.sh` | 跨节点连通性验证 + 端口暴露检查 |
| `tailscale/acls.hujson` | Tailscale ACL 策略 (粘贴到 Admin Console) |

**ACL 策略** (详见 `tailscale/acls.hujson`)：

```json
{
  "tagOwners": {
    "tag:quant-master": ["autogroup:admin"],
    "tag:quant-yf":     ["autogroup:admin"],
    "tag:quant-cn":     ["autogroup:admin"]
  },
  "acls": [
    {"action": "accept", "src": ["tag:quant-master", "tag:quant-yf", "tag:quant-cn"],
     "dst": ["tag:quant-master:*", "tag:quant-yf:*", "tag:quant-cn:*"]},
    {"action": "accept", "src": ["tag:quant-master"], "dst": ["tag:quant-yf:8000"]},
    {"action": "accept", "src": ["tag:quant-cn"], "dst": ["tag:quant-master:6379"]}
  ]
}
```

- 子服务 **8000 不对公网开放**（防火墙 drop）；只监听 Tailscale IP 或 `tailscale0`。  
- Redis/Postgres **禁止** `0.0.0.0` 对公网。  
- SSH 优先 Tailscale SSH，收紧公网 22。

### 5. Git 分支策略与 CI/CD 触发规则

**分支保护规则**（GitHub Settings → Branches → main）：
- ✅ **Require a pull request before merging** — 禁止直接 push 到 main
- ✅ **Require status checks to pass** — CI 必须通过才能合并
  - `backend-ci / lint`
  - `backend-ci / test`
  - `backend-ci / security`
- ✅ **Do not allow force pushes** — 禁止强制推送
- ✅ **Do not allow deletions** — 禁止删除分支
- ✅ **Do not allow bypassing** — 管理员也不绕过

**CI/CD 触发规则**：

| 事件 | lint | test | security | 🚀 部署 |
|------|:---:|:---:|:---:|:---:|
| push `develop` | ✅ | ✅ | ✅ | ❌ |

## 二、Cloudflare 免费资源完整利用方案

### 2.1 Cloudflare Pages（前端 SPA 托管）

**零费用替代 Nginx 静态服务**，全球 CDN 节点，国内访问速度显著优于 VPS 直连：

```
# .github/workflows/deploy-frontend.yml

on:
  push:
    branches: [main]
    paths: ['frontend/**']

jobs:
  deploy-pages:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'pnpm' }
      - run: pnpm install && pnpm build
        working-directory: frontend
        env:
          VITE_API_BASE_URL: ${{ secrets.API_BASE_URL }}   # Tailscale 内网 IP 或公网域名
          VITE_WS_BASE_URL:  ${{ secrets.WS_BASE_URL }}
      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CF_API_TOKEN }}
          accountId: ${{ secrets.CF_ACCOUNT_ID }}
          command: pages deploy frontend/dist --project-name=quant-edge
```

**Cloudflare Pages 配置**：
```
项目名：quant-edge
构建命令：pnpm build（GitHub Actions 负责，不用 CF 内置构建）
部署目录：frontend/dist
自定义域名：app.yourdomain.com（绑定后 SSL 自动签发）
```

### 2.4 Cloudflare R2（对象存储）

**完全替代 AWS S3**，免费额度：10GB 存储 + 100万次 Class A 操作 / 月，存储研报 PDF、回测结果等：

```python
# backend/services/r2_storage.py — 使用 boto3（S3 兼容 API）

import boto3
from botocore.config import Config

r2_client = boto3.client(
    's3',
    endpoint_url=f'https://{CF_ACCOUNT_ID}.r2.cloudflarestorage.com',
    aws_access_key_id=CF_R2_ACCESS_KEY,
    aws_secret_access_key=CF_R2_SECRET_KEY,
    config=Config(signature_version='s3v4'),
    region_name='auto',
)

async def upload_backtest_report(report_id: str, content: bytes) -> str:
    key = f"reports/{report_id}.html"
    r2_client.put_object(
        Bucket='quant-reports',
        Key=key,
        Body=content,
        ContentType='text/html',
    )
    return f"https://reports.yourdomain.com/{key}"  # R2 自定义域名
```

**R2 存储分类**：
```
quant-reports/
  ├── backtest/{strategy_id}/{timestamp}.html    回测撕裂报告
  ├── research/{company}/{date}.pdf              研报 PDF
  └── exports/{user_id}/{timestamp}.csv          持仓导出

quant-backups/
  ├── redis/{date}/dump.rdb                      Redis 每日备份
  └── pgvector/{date}/vector_export.pgdump       向量库备份
```

### 2.5 Cloudflare Workers（边缘缓存 / 轻量 API）

**适合：API 缓存代理、宏观日历静态数据缓存**（减少 VPS 请求压力）：

```javascript
// cloudflare-workers/macro-cache-worker.js
// 部署：wrangler deploy — 全球边缘节点，响应 < 10ms

export default {
  async fetch(request, env) {
    const url = new URL(request.url)

    // 宏观日历：每小时缓存
    if (url.pathname === '/macro/calendar') {
      const cached = await env.MACRO_KV.get('calendar', 'json')
      if (cached) {
        return Response.json(cached, {
          headers: { 'X-Cache': 'HIT', 'Cache-Control': 'max-age=3600' }
        })
      }
      // 穿透到 VPS
      const upstream = await fetch(`https://api.yourdomain.com${url.pathname}`, {
        headers: { 'Authorization': request.headers.get('Authorization') }
      })
      const data = await upstream.json()
      await env.MACRO_KV.put('calendar', JSON.stringify(data), { expirationTtl: 3600 })
      return Response.json(data, { headers: { 'X-Cache': 'MISS' } })
    }

    return fetch(request)
  }
}
```

## 四之一、跨节点数据源路由与 YFinance 多 VPS 抗限流

> 实现指针：已落地 `ServiceRegistry`（DIST-01）+ `YFinanceRouter`（DIST-02/04）+ `data_subservice`（DIST-05~10）。本节定义 **四节点运营规则**，取代 V8.0「单备用 + 公网 URL」叙述。

### 4.5 YFinance 多 VPS 获取策略（抗 429）

**问题**：Yahoo 对 **出口 IP** 限流；主节点直连打满后，API/Worker 整机抖动。

**策略（必须）**：

```
请求 → YFinanceRouter
         │
         ├─ discover(ServiceRegistry, capability=yfinance)
         │     健康 ∧ 未 throttled ∧ 未熔断
         │
         ├─ 排序：未限流优先 → weight 降序 → 连续 429 次数升序
         │
         ├─ 选 US-YF-A 或 US-YF-B（加权轮询）
         │     HMAC + Tailscale HTTP
         │
         ├─ 若 429 / rate_limit：
         │     不计熔断失败计数（见 docs/14 §10.8）
         │     触发该节点 RateLimitThrottler 退避
         │     failover 下一节点（同请求不重打同一 IP）
         │
         └─ 全部不可用 → Redis STALE 缓存；禁止疯狂重试 Yahoo
```

**节点配额与权重建议**：

| 节点 | weight | 说明 |
|:---|:---:|:---|
| US-YF-A | 10 | 主流量 |
| US-YF-B | 10 | 对等分担（勿设成「热备 0 流量」——两 IP 都要热起来才有配额） |
| US-MASTER 本地 YF | 0~1 | **默认 0（关闭）**；仅紧急兜底 |

**环境变量（US-MASTER）**：

```bash
YF_ROUTER_ENABLED=true
DATASOURCE_YFINANCE_MODE=external   # 或 hybrid 且本地 weight=0

# 发现走 Registry（推荐），无需手写死 IP；若静态兜底：
# YF_STATIC_NODES=http://us-yf-a:8000,http://us-yf-b:8000
DATA_SOURCE_HMAC_SECRET=<vault>
REDIS_URL=redis://127.0.0.1:6379/0
```

**环境变量（US-YF-A / US-YF-B 相同模板）**：

```bash
NODE_ID=us-yf-a                      # B 节点改为 us-yf-b
DS_SOURCES_ENABLED=yfinance
REGISTRY_REDIS_URL=redis://us-master:6379/0   # Tailscale DNS
REGISTRY_CAPABILITIES=yfinance,quote,history,macro,batch_quote
NODE_WEIGHT=10
DS_BIND_HOST=100.x.x.x               # 仅绑 Tailscale IP
DATA_SOURCE_HMAC_SECRET=<same>
# 可选：每节点软限流，保护本 IP
DATASOURCE_YFINANCE_BACKOFF_STRATEGY=exponential
```

**运营纪律**：

1. **两辅助节点必须不同公网 IP / 不同云账号出口**（同机双容器无效）。  
2. 宏观 daemon / 批量 sync **打散到两节点**（Router 自然分担）；禁止 cron 钉死单一 NODE_ID。  
3. 观察到单节点 `consecutive_rate_limits` 上升 → 临时降 weight，勿在主节点开本地硬重试。  
4. 扩容：加 **US-YF-C** 同模板即可，Registry 自动发现（DIST 热插拔）。

### 6.1 表结构设计

```
-- PostgreSQL + pgvector 向量存储表

CREATE EXTENSION IF NOT EXISTS vector;

-- 知识库文档块（RAG 检索）
CREATE TABLE knowledge_chunks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_url  TEXT NOT NULL,
    title       TEXT,
    content     TEXT NOT NULL,
    embedding   vector(1536),           -- OpenAI text-embedding-3-small 维度
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    updated_at  TIMESTAMPTZ DEFAULT NOW(),
    expires_at  TIMESTAMPTZ,            -- NULL = 永不过期；有值 = 定期清理过期研报
    metadata    JSONB DEFAULT '{}'
);

-- 向量相似度索引（IVFFlat，平衡速度与精度）
CREATE INDEX idx_knowledge_embedding
ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);                     -- lists = sqrt(行数)，按需调整

-- 全文检索索引（混合检索：向量 + 关键词）
CREATE INDEX idx_knowledge_content_fts
ON knowledge_chunks USING gin(to_tsvector('english', content));
```

### 6.2 向量查询（混合检索）

```python
# backend/services/knowledge_service.py

async def hybrid_search(
    query: str,
    query_embedding: list[float],
    days_back: int | None = None,
    top_k: int = 10,
) -> list[KnowledgeChunk]:
    """向量相似度 + 关键词的混合检索"""

    expire_filter = ""
    if days_back:
        expire_filter = f"AND created_at > NOW() - INTERVAL '{days_back} days'"

    sql = """
    SELECT id, content, source_url, title,
           1 - (embedding <=> $1::vector) AS similarity,   -- 余弦相似度
           ts_rank(to_tsvector('english', content), plainto_tsquery($2)) AS text_rank
    FROM knowledge_chunks
    WHERE expires_at IS NULL OR expires_at > NOW()
    {expire_filter}
    ORDER BY
      (1 - (embedding <=> $1::vector)) * 0.7 +            -- 向量权重 70%
      ts_rank(to_tsvector('english', content), plainto_tsquery($2)) * 0.3  -- 关键词 30%
    DESC
    LIMIT $3
    """.format(expire_filter=expire_filter)

    return await db.fetch(sql, query_embedding, query, top_k)
```

### 7.2 外部 API（YFinance / FRED / Finnhub）稳定抓取

所有外部数据源直接在主服务中运行，通过熔断器 + 重试 + Redis 缓存三层防护：

```python
# backend/services/external_fetcher.py

class ExternalDataFetcher:
    """外部 API 熔断 + 重试 + Redis 缓存三层防护"""

    def __init__(self, redis: Redis):
        self.redis = redis
        self._circuit_breakers: dict[str, CircuitBreaker] = {}

    async def fetch_with_cache(
        self,
        cache_key: str,
        fetch_fn: Callable,
        ttl: int = 300,
        circuit_name: str = "default",
    ) -> Any:
        cached = await self.redis.get(cache_key)
        if cached:
            return json.loads(cached)

        cb = self._circuit_breakers.setdefault(
            circuit_name, CircuitBreaker(fail_max=3, reset_timeout=60)
        )

        try:
            with cb:
                data = await asyncio.to_thread(fetch_fn)
                await self.redis.setex(cache_key, ttl, json.dumps(data))
                return data
        except CircuitBreakerError:
            logger.warning('circuit_open', source=circuit_name)
            stale = await self.redis.get(f"stale:{cache_key}")
            return {"data": json.loads(stale), "stale": True} if stale else None
        except Exception as e:
            logger.error('fetch_failed', source=circuit_name, error=str(e))
            await self.redis.rename(cache_key, f"stale:{cache_key}")
            raise
```

### 8.1 Compose 文件 / Profile

| 产物 | 节点 | 内容 |
|:---|:---|:---|
| `docker-compose.master.yml` | US-MASTER | redis · postgres · api · worker ·（monitoring profile） |
| `docker-compose.yf-node.yml` | US-YF-A / US-YF-B | 仅 `data_subservice`（yfinance） |
| `docker-compose.slave.yml` | CN-AKSHARE | AKShare collector / 薄 worker |
| `docker-compose.yml` | 本地开发 | 单机全量（YF Router 可关） |

> 若仓库尚未拆出 `yf-node.yml`，短期可用 `data_subservice` Dockerfile + 独立 compose 覆盖；**DIST-11** 跟踪编排落地。

### 8.2 US-MASTER

```bash
ssh us-master   # Tailscale SSH
cd /opt/quant_agent && git pull
# .env：YF_ROUTER_ENABLED=true · COLLECTOR_YFINANCE=false · REDIS/PG 仅本地
docker compose -f docker-compose.master.yml --profile monitoring up -d
curl -f http://127.0.0.1:8000/api/v1/health
```

Futu OpenD：**宿主机** systemd，监听 `127.0.0.1:11111`；容器 `extra_hosts: host.docker.internal:host-gateway`。

### 8.3 US-YF-A / US-YF-B（对称部署）

```bash
# 在两台不同公网 IP 的美国 VPS 上各执行一次
ssh us-yf-a
cd /opt/quant_agent && git pull
cp .env.yf-node.example .env
# NODE_ID=us-yf-a · REGISTRY_REDIS_URL=redis://us-master:6379/0 · NODE_WEIGHT=10
docker compose -f docker-compose.yf-node.yml up -d
# 仅健康检查走 Tailscale；公网 UFW deny 8000
```

US-YF-B：复制流程，改 `NODE_ID=us-yf-b`（及独立 `.env`）。

### 8.4 CN-AKSHARE

```bash
ssh cn-akshare
cd /opt/quant_agent && git pull
# REDIS_HOST=us-master · COLLECTOR_AKSHARE=true · 关闭 YF/Futu
docker compose -f docker-compose.slave.yml up -d
```

### 8.5 验收清单（上线门禁）

```
□ tailscale status 四节点 online
□ Registry 中 yfinance 实例数 ≥ 2，且 node_id 不同
□ 压测 batch_quote：429 时流量切到另一 US-YF，主 API 不 5xx
□ COLLECTOR_YFINANCE 在 MASTER 为 false（或 weight=0）
□ CN 节点进程列表无 yfinance
□ 公网 nmap：YF 节点 8000 / Redis 6379 对公网 closed
□ Cloudflare Pages API 仅指向 US-MASTER
```

## 九、安全加固清单

### 9.1 端口管理（最小暴露原则）

```
Tailscale 模式（推荐）：
  VPS 防火墙：所有入站端口关闭，仅允许 Tailscale 内网 IP 访问
  SSH 通过 tailscale ssh 替代公网 SSH，无需开放 22 端口
  跨节点通信走 Tailscale P2P 直连，无需公网暴露
  前端 API 通过 Cloudflare Pages 对外提供服务

备用 Nginx 模式（无 Tailscale）：
  开放：80, 443（Nginx）
  开放：22（SSH，改为 2222 或其他非标端口）
  禁止：所有其他端口
```

### 9.2 SSH 加固

```bash
# /etc/ssh/sshd_config
Port 2222                    # 非标准端口，避免机器人扫描
PermitRootLogin no
PasswordAuthentication no    # 只允许 SSH Key 登录
MaxAuthTries 3
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
```

### 9.3 敏感配置管理

```bash
# .env（不提交 git，VPS 独立配置）
# GitHub Actions 的 Secrets 中单独管理

# === 单一 VPS 配置 ===
SECRET_KEY=          # JWT 签名密钥（openssl rand -hex 32 生成）
REDIS_PASSWORD=      # Redis 密码（强密码，≥ 32 字符）
DB_PASSWORD=         # PostgreSQL 密码
OPENAI_API_KEY=      # OpenAI API Key
CF_API_TOKEN=        # Cloudflare API Token（仅 Pages/R2 权限）
CF_R2_ACCESS_KEY=    # R2 Access Key
CF_R2_SECRET_KEY=    # R2 Secret Key
GRAFANA_PASSWORD=    # Grafana admin 密码

# 采集器开关
COLLECTOR_FUTU=true
COLLECTOR_YFINANCE=true
COLLECTOR_FINNHUB=false
COLLECTOR_AKSHARE=false

# Futu OpenD 配置（宿主机本地）
FUTU_HOST=127.0.0.1
FUTU_PORT=11111
FUTU_TRADE_PASSWORD= # 富途交易解锁密码

# Finnhub API Key（启用 COLLECTOR_FINNHUB 时需要）
FINNHUB_API_KEY=

# === 跨节点数据源路由配置 ===
DATA_SOURCE_ROUTER_ENABLED=false    # 是否启用跨节点数据源路由
DATA_SOURCE_HMAC_SECRET=            # 节点间通信签名密钥（所有节点必须一致）

# YFinance 多源切换（防止限流）
YF_PRIMARY_NODE_URL=http://localhost:8000   # YFinance 主节点
YF_BACKUP_NODE_URL=                         # YFinance 备用节点（可选）

# AKShare 远程节点（国内 VPS）
AKSHARE_REMOTE_URL=                         # AKShare 远程节点地址（可选）
```
