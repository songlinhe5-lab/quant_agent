## 一、全局部署拓扑设计

### 1.1 节点规划原则

个人量化系统的部署面临三个核心约束：
1. **akshare 等国内数据源必须从境内直连**（境外 IP 无法访问或被限速）
2. **yfinance / finnhub / futuopenai 等海外数据源需境外 IP**（境内 IP 被封或严重限速）
3. **Web 前端需要对全球用户（包括中国大陆）快速访问**，通过 Cloudflare Pages CDN 分发

基于以上约束，确定以下节点分工：

**服务注册协议**：

```python
# 加州子服务启动时注册
class DataServiceRegistry:
    """子服务注册与发现"""

    async def register(self, node_id: str, host: str, port: int, capabilities: list[str]):
        """注册子服务节点"""
        payload = {
            "node_id": node_id,
            "host": host,
            "port": port,
            "capabilities": capabilities,  # ["yfinance", "finnhub", "futuopenai"]
            "registered_at": datetime.utcnow().isoformat(),
            "status": "active",
        }
        # 写入 Redis Hash，TTL 120s（心跳续期）
        await self.redis.hset(f"registry:node:{node_id}", mapping=payload)
        await self.redis.expire(f"registry:node:{node_id}", 120)

    async def heartbeat(self, node_id: str):
        """心跳续期（每 30s 调用一次）"""
        await self.redis.expire(f"registry:node:{node_id}", 120)
        await self.redis.hset(f"registry:node:{node_id}", "last_heartbeat", datetime.utcnow().isoformat())

    async def discover(self, capability: str) -> list[dict]:
        """根据能力发现可用节点"""
        nodes = []
        async for key in self.redis.scan_iter("registry:node:*"):
            info = await self.redis.hgetall(key)
            if capability in info.get("capabilities", "[]"):
                nodes.append(info)
        return nodes
```

**主服务动态路由**：

```python
# backend/core/service_router.py

class ServiceRouter:
    """根据数据源类型动态路由至对应节点"""

    async def route_request(self, data_source: str, params: dict) -> dict:
        """路由数据请求到合适的节点"""
        # 国内数据源 → 本地直接处理
        if data_source in ("akshare", "a_stock", "cn_macro"):
            return await self._local_fetch(data_source, params)

        # 海外数据源 → 查找加州子服务
        nodes = await self.registry.discover(data_source)
        if not nodes:
            raise ServiceUnavailableError(f"No node available for {data_source}")

        # 负载均衡（当前单节点，后续可扩展）
        node = nodes[0]
        url = f"https://{node['host']}:{node['port']}/api/v1/data/{data_source}"
        async with httpx.AsyncClient() as client:
            resp = await client.get(url, params=params, timeout=30)
            return resp.json()
```

## 二、Cloudflare 免费资源完整利用方案

### 2.1 Cloudflare Pages（前端 SPA 托管）

**零费用替代 Nginx 静态服务**，全球 CDN 节点，国内访问速度显著优于 VPS 直连：

```yaml
# .github/workflows/deploy-frontend.yml

on:
  push:
    branches: [main]
    paths: ['frontend/**']

jobs:
  deploy-pages:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'pnpm' }
      - run: pnpm install && pnpm build
        working-directory: frontend
        env:
          VITE_API_BASE_URL: ${{ secrets.API_BASE_URL }}   # Cloudflare Tunnel 地址
          VITE_WS_BASE_URL:  ${{ secrets.WS_BASE_URL }}
      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CF_API_TOKEN }}
          accountId: ${{ secrets.CF_ACCOUNT_ID }}
          command: pages deploy frontend/dist --project-name=quant-edge
```

**Cloudflare Pages 配置**：
```
项目名：quant-edge
构建命令：pnpm build（GitHub Actions 负责，不用 CF 内置构建）
部署目录：frontend/dist
自定义域名：app.yourdomain.com（绑定后 SSL 自动签发）
```

### 2.2 Cloudflare Tunnel（零信任安全接入）

**核心价值**：北京 VPS 无需开放任何公网端口（无 22/8000/3001/9090），彻底消灭端口扫描攻击面。所有流量通过 Cloudflare 加密隧道转发：

```bash
# 在节点 A（北京 VPS）安装 cloudflared
wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
dpkg -i cloudflared-linux-amd64.deb

# 认证并创建 Tunnel
cloudflared tunnel login
cloudflared tunnel create quant-core

# 配置路由（/etc/cloudflared/config.yml）
tunnel: <TUNNEL_UUID>
credentials-file: /root/.cloudflared/<TUNNEL_UUID>.json

ingress:
  # API 网关（FastAPI WebSocket + REST）
  - hostname: api.yourdomain.com
    service: http://localhost:8000
  # Grafana 监控面板（配合 CF Access 鉴权）
  - hostname: grafana.yourdomain.com
    service: http://localhost:3001
  # 捕获所有其他请求
  - service: http_status:404

# 注册为系统服务（开机自启）
cloudflared service install
```

**WebSocket 特殊配置**（行情推送）：
```yaml
# Tunnel 默认支持 WebSocket，但需设置超时
ingress:
  - hostname: api.yourdomain.com
    service: http://localhost:8000
    originRequest:
      connectTimeout: 30s
      noTLSVerify: false
      httpHostHeader: "api.yourdomain.com"
      # WebSocket 长连接保活
      keepAliveConnections: 100
      keepAliveTimeout: 90s
```

**跨节点通信配置**（北京 ↔ 加州）：
```yaml
# 加州子服务通过 Cloudflare Tunnel 访问北京主服务
# 无需开放北京 VPS 公网端口，Tunnel 提供加密隧道
ingress:
  # 内部 API（仅供加州子服务调用）
  - hostname: internal.yourdomain.com
    service: http://localhost:8000
    originRequest:
      # HMAC 签名验证，防止未授权访问
      httpHostHeader: "internal.yourdomain.com"
```

### 2.4 Cloudflare R2（对象存储）

**完全替代 AWS S3**，免费额度：10GB 存储 + 100万次 Class A 操作 / 月，存储研报 PDF、回测结果等：

```python
# backend/services/r2_storage.py — 使用 boto3（S3 兼容 API）

import boto3
from botocore.config import Config

r2_client = boto3.client(
    's3',
    endpoint_url=f'https://{CF_ACCOUNT_ID}.r2.cloudflarestorage.com',
    aws_access_key_id=CF_R2_ACCESS_KEY,
    aws_secret_access_key=CF_R2_SECRET_KEY,
    config=Config(signature_version='s3v4'),
    region_name='auto',

async def upload_backtest_report(report_id: str, content: bytes) -> str:
    key = f"reports/{report_id}.html"
    r2_client.put_object(
        Bucket='quant-reports',
        Key=key,
        Body=content,
        ContentType='text/html',
    )
    return f"https://reports.yourdomain.com/{key}"  # R2 自定义域名
```

### 2.5 Cloudflare Workers（边缘缓存 / 轻量 API）

**适合：API 缓存代理、宏观日历静态数据缓存**（减少 VPS 请求压力）：

```javascript
// cloudflare-workers/macro-cache-worker.js
// 部署：wrangler deploy — 全球边缘节点，响应 < 10ms

export default {
  async fetch(request, env) {
    const url = new URL(request.url)

    // 宏观日历：每小时缓存
    if (url.pathname === '/macro/calendar') {
      const cached = await env.MACRO_KV.get('calendar', 'json')
      if (cached) {
        return Response.json(cached, {
          headers: { 'X-Cache': 'HIT', 'Cache-Control': 'max-age=3600' }
        })
      }
      // 穿透到 VPS
      const upstream = await fetch(`https://api.yourdomain.com${url.pathname}`, {
        headers: { 'Authorization': request.headers.get('Authorization') }
      })
      const data = await upstream.json()
      await env.MACRO_KV.put('calendar', JSON.stringify(data), { expirationTtl: 3600 })
      return Response.json(data, { headers: { 'X-Cache': 'MISS' } })
    }

    return fetch(request)
  }
}
```

## 三、GitHub 免费资源完整利用方案

### 3.1 GitHub Container Registry（Docker 镜像仓库）

**替代 Docker Hub**，私有镜像免费，无速率限制：

```yaml
# .github/workflows/build-push.yml

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}   # ghcr.io/yourusername/quant_agent

jobs:
  build-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - uses: docker/metadata-action@v5
        id: meta
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest,enable=${{ github.ref == 'refs/heads/main' }}

      - uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha        # GitHub Actions 缓存层 — 加快构建
          cache-to: type=gha,mode=max
```

### 3.2 GitHub Actions CI/CD 完整流水线

```
代码推送 → 质量检查 → 构建镜像 → 自动部署 → 通知
```

```yaml
# .github/workflows/ci-cd.yml — 完整流水线

name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  # ── 质量门禁 ──────────────────────────────────────────
  quality-gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
      - run: uv sync --all-extras
      - run: uv run ruff check && uv run ruff format --check
      - run: uv run pytest tests/ -x --tb=short -q
        env:
          REDIS_HOST: localhost        # CI 环境用本地 Redis
          DATABASE_URL: sqlite:///./test.db

  # ── 前端构建 + Cloudflare Pages 部署 ─────────────────
  deploy-frontend:
    needs: quality-gate
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - run: pnpm install && pnpm build
        working-directory: frontend
        env:
          VITE_API_BASE_URL: ${{ secrets.API_BASE_URL }}
      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CF_API_TOKEN }}
          command: pages deploy frontend/dist --project-name=quant-edge

  # ── 构建并推送 Docker 镜像 ─────────────────────────────
  build-backend:
    needs: quality-gate
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions: { contents: read, packages: write }
    outputs:
      image_tag: ${{ steps.meta.outputs.version }}
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/metadata-action@v5
        id: meta
        with:
          images: ghcr.io/${{ github.repository }}
          tags: type=sha,prefix=sha-
      - uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ── SSH 部署至节点 A（香港 VPS）─────────────────────────
  deploy-hk:
    needs: build-backend
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    environment: production-hk       # 需要手动审批（保护生产环境）
    steps:
      - uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.HK_VPS_HOST }}
          username: ${{ secrets.HK_VPS_USER }}
          key: ${{ secrets.HK_VPS_SSH_KEY }}
          script: |
            cd ~/quant_agent
            git pull origin main
            echo ${{ secrets.GITHUB_TOKEN }} | docker login ghcr.io -u ${{ github.actor }} --password-stdin
            docker pull ghcr.io/${{ github.repository }}:${{ needs.build-backend.outputs.image_tag }}
            docker-compose up -d --no-build
            docker system prune -f   # 清理旧镜像

  # ── Telegram / 钉钉 部署通知 ──────────────────────────
  notify:
    needs: [deploy-frontend, deploy-hk]
    if: always()
    runs-on: ubuntu-latest
    steps:
      - uses: appleboy/telegram-action@master
        with:
          to: ${{ secrets.TELEGRAM_CHAT_ID }}
          token: ${{ secrets.TELEGRAM_TOKEN }}
          message: |
            🚀 QuantEdge 部署完成
            分支: ${{ github.ref_name }}
            前端: ${{ needs.deploy-frontend.result }}
            后端: ${{ needs.deploy-hk.result }}
```

## 四、数据子服务稳定运行方案（加州节点）

### 4.1 数据子服务部署配置

加州 VPS 运行数据子服务容器，采集海外数据源（yfinance / finnhub / futuopenai），启动时向北京主服务注册：

```yaml

version: '3.8'

services:
  data-subservice:
    image: ghcr.io/${GITHUB_REPO}/data-subservice:latest
    container_name: quant_data_sub
    restart: always
    command: >
      uvicorn data_subservice.main:app
      --host 0.0.0.0 --port 8100
      --workers 2
      --loop uvloop --http httptools
    env_file: .env.node-b
    environment:
      # 注册到北京主服务
      MAIN_SERVICE_URL: ${MAIN_SERVICE_URL}  # https://internal.yourdomain.com

      FINNHUB_API_KEY: ${FINNHUB_API_KEY}
      FUTU_OPEND_HOST: ${FUTU_OPEND_HOST}
      FUTU_OPEND_PORT: ${FUTU_OPEND_PORT}
    expose:
      - "8100"
    healthcheck:
```

### 4.2 子服务注册与心跳机制

```python
# data_subservice/registry_client.py

import asyncio
import httpx
import time

class RegistryClient:
    """向北京主服务注册并维持心跳"""

    def __init__(self, main_service_url: str, node_id: str, hmac_secret: str):
        self.main_url = main_service_url
        self.node_id = node_id
        self.hmac_secret = hmac_secret
        self._running = True

    async def register(self, host: str, port: int, capabilities: list[str]):
        """启动时注册"""
        payload = {
            "node_id": self.node_id,
            "host": host,
            "port": port,
            "capabilities": capabilities,
        }
        headers = self._sign_request(payload)
        async with httpx.AsyncClient() as client:
            resp = await client.post(
                f"{self.main_url}/api/v1/internal/registry/register",
                json=payload,
                headers=headers,
                timeout=10,
            )
            resp.raise_for_status()

    async def heartbeat_loop(self, interval: int = 30):
        """心跳保活循环（每 30s 发送一次）"""
        while self._running:
            try:
                headers = self._sign_request({"node_id": self.node_id})
                async with httpx.AsyncClient() as client:
                    await client.post(
                        f"{self.main_url}/api/v1/internal/registry/heartbeat",
                        json={"node_id": self.node_id, "ts": time.time()},
                        headers=headers,
                        timeout=10,
                    )
            except Exception as e:
                logger.warning(f"Heartbeat failed: {e}, will retry")
            await asyncio.sleep(interval)

    def _sign_request(self, payload: dict) -> dict:
        """HMAC-SHA256 签名"""
        import hashlib, hmac, json
        body = json.dumps(payload, sort_keys=True)
        sig = hmac.new(self.hmac_secret.encode(), body.encode(), hashlib.sha256).hexdigest()
        return {"X-Internal-Sig": sig, "X-Node-ID": self.node_id}
```

### 6.1 表结构设计

```sql
-- PostgreSQL + pgvector 向量存储表

CREATE EXTENSION IF NOT EXISTS vector;

-- 知识库文档块（RAG 检索）
CREATE TABLE knowledge_chunks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_url  TEXT NOT NULL,
    title       TEXT,
    content     TEXT NOT NULL,
    embedding   vector(1536),           -- OpenAI text-embedding-3-small 维度
    created_at  TIMESTAMPTZ DEFAULT NOW(),
    updated_at  TIMESTAMPTZ DEFAULT NOW(),
    expires_at  TIMESTAMPTZ,            -- NULL = 永不过期；有值 = 定期清理过期研报
    metadata    JSONB DEFAULT '{}'
);

-- 向量相似度索引（IVFFlat，平衡速度与精度）
CREATE INDEX idx_knowledge_embedding
ON knowledge_chunks USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);                     -- lists = sqrt(行数)，按需调整

-- 全文检索索引（混合检索：向量 + 关键词）
CREATE INDEX idx_knowledge_content_fts
ON knowledge_chunks USING gin(to_tsvector('english', content));
```

### 6.2 向量查询（混合检索）

```python
# backend/services/knowledge_service.py

async def hybrid_search(
    query: str,
    query_embedding: list[float],
    days_back: int | None = None,
    top_k: int = 10,
) -> list[KnowledgeChunk]:
    """向量相似度 + 关键词的混合检索"""

    expire_filter = ""
    if days_back:
        expire_filter = f"AND created_at > NOW() - INTERVAL '{days_back} days'"

    sql = """
    SELECT id, content, source_url, title,
           1 - (embedding <=> $1::vector) AS similarity,   -- 余弦相似度
           ts_rank(to_tsvector('english', content), plainto_tsquery($2)) AS text_rank
    FROM knowledge_chunks
    WHERE expires_at IS NULL OR expires_at > NOW()
    {expire_filter}
    ORDER BY
      (1 - (embedding <=> $1::vector)) * 0.7 +            -- 向量权重 70%
      ts_rank(to_tsvector('english', content), plainto_tsquery($2)) * 0.3  -- 关键词 30%
    DESC
    LIMIT $3
    """.format(expire_filter=expire_filter)

    return await db.fetch(sql, query_embedding, query, top_k)
```

> **架构变更**：YFinance / Finnhub / FutuOpenAI 等海外数据源已迁移至加州子服务节点（节点 B），北京主服务通过服务注册表动态路由。

```python
# data_subservice/fetcher.py（运行在加州节点）

class ExternalDataFetcher:
    """外部 API 熔断 + 重试 + Redis 缓存三层防护"""

    def __init__(self, redis: Redis):
        self.redis = redis
        self._circuit_breakers: dict[str, CircuitBreaker] = {}

    async def fetch_with_cache(
        self,
        cache_key: str,
        fetch_fn: Callable,
        ttl: int = 300,
        circuit_name: str = "default",
    ) -> Any:
        # 1. 先查 Redis 缓存
        cached = await self.redis.get(cache_key)
        if cached:
            return json.loads(cached)

        # 2. 熔断器检查（连续失败 3 次则熔断 60s）
        cb = self._circuit_breakers.setdefault(
            circuit_name, CircuitBreaker(fail_max=3, reset_timeout=60)
        )

        try:
            with cb:
                data = await asyncio.to_thread(fetch_fn)
                await self.redis.setex(cache_key, ttl, json.dumps(data))
                return data
        except CircuitBreakerError:
            logger.warning('circuit_open', source=circuit_name)
            stale = await self.redis.get(f"stale:{cache_key}")
            return {"data": json.loads(stale), "stale": True} if stale else None
        except Exception as e:
            logger.error('fetch_failed', source=circuit_name, error=str(e))
            await self.redis.rename(cache_key, f"stale:{cache_key}")
            raise
```
