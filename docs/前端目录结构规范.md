# Frontend 目录结构规范

> **版本**: V1.0 | **日期**: 2026-06-27  
> **基于**: ADR-001 (Pure Vite SPA) + React Router v6

## 标准目录结构

```
frontend/src/
├── main.tsx                      # 应用入口
├── App.tsx                       # 根组件
│
├── components/                   # 共享 UI 组件
│   ├── ui/                      # shadcn/ui 基础组件
│   ├── layout/                  # 布局组件 (Navbar, Sidebar, StatusBar)
│   └── feedback/                # 反馈组件 (ErrorBoundary, Toast)
│
├── features/                     # 功能模块 (按业务领域)
│   ├── quotes/                  # 行情
│   ├── screener/                # 选股
│   ├── strategy/                # 策略 IDE
│   ├── backtest/                # 回测
│   ├── oms/                     # 订单管理
│   ├── risk/                    # 风控
│   ├── copilot/                 # AI 副驾
│   ├── data-center/             # 数据中心
│   ├── settings/                # 设置
│   ├── apm/                     # APM 监控
│   └── auth/                    # 认证
│
├── router/                       # React Router 配置
│   └── index.tsx
│
├── stores/                       # Zustand 全局 Store
│   ├── use-watchlist.ts
│   ├── useSystemStore.ts
│   └── marketStore.ts
│
├── hooks/                        # 自定义 Hooks
│   ├── use-market-data.ts
│   ├── use-api.ts
│   └── use-toast.ts
│
├── services/                     # API 服务层
│   ├── api-client.ts            # HTTP 客户端
│   ├── ws-client.ts             # WebSocket 客户端
│   └── mock.ts                  # Mock 数据
│
├── types/                        # TypeScript 类型定义
│   ├── domain.ts                # 领域对象 (Quote, Kline, Order)
│   └── api.ts                   # API 请求/响应类型
│
├── lib/                          # 工具函数库
│   ├── utils.ts                 # 通用工具
│   └── constants.ts             # 常量定义
│
├── workers/                      # Web Workers
│   └── indicator-worker.ts
│
├── styles/                       # 全局样式
│   └── globals.css
│
├── locales/                      # 国际化
│   ├── zh-CN.json
│   └── en-US.json
│
├── contexts/                     # React Contexts (逐步迁移到 hooks)
└── assets/                       # 静态资源
    ├── images/
    └── icons/
```

## 功能模块标准结构

每个功能模块 (如 `features/quotes/`) 应遵循以下结构：

```
features/quotes/
├── index.tsx                    # 模块主入口
├── types.ts                     # 模块类型定义
├── components/                  # 模块专有组件
│   ├── watchlist-sidebar.tsx
│   ├── lightweight-chart-canvas.tsx
│   ├── order-book.tsx
│   └── ...
├── hooks/                       # 模块专有 hooks
│   └── use-quotes-stream.ts
├── stores/                      # 模块专有 store (如需要)
│   └── use-quotes-store.ts
└── utils/                       # 模块专有工具函数
    └── chart-helpers.ts
```

## 命名规范

1. **文件命名**: kebab-case (如 `use-market-data.ts`)
2. **组件命名**: PascalCase (如 `WatchlistSidebar`)
3. **Hook 命名**: camelCase with `use` prefix (如 `useMarketData`)
4. **类型命名**: PascalCase with `T` prefix or suffix (如 `QuoteT` 或 `type Quote`)

## 需要删除的目录

- ❌ `app/` - Next.js App Router 残留
- ❌ `views/` - 功能已迁移到 `features/`

## 需要创建的文件

- ✅ `services/api-client.ts` - HTTP 客户端
- ✅ `services/ws-client.ts` - WebSocket 客户端
- ✅ `types/domain.ts` - 领域类型定义
- ✅ `lib/constants.ts` - 常量定义

## 迁移步骤

1. 删除 `app/` 目录
2. 将 `views/` 中的内容合并到 `features/`
3. 创建缺失的 `services/`, `types/`, `lib/` 文件
4. 更新 `router/index.tsx` 指向新的 `features/` 模块
5. 运行 `pnpm dlx @microsoft/api-extractor` 验证类型导出
