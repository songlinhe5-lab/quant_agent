# Frontend 目录结构规范

> **版本**: V1.1 | **日期**: 2026-07-13  
> **基于**: ADR-001 (Pure Vite SPA) + React Router v6  
> **架构 SSOT**: `docs/04. 前端架构与零GC渲染.md` V4.0 · 铁律：`.cursor/rules/vibe-coding.mdc`

## 标准目录结构

```
frontend/src/
├── main.tsx                      # 应用入口
├── App.tsx                       # 路由 SSOT（禁止平行 router/index）
│
├── components/
│   ├── ui/                       # shadcn 原子组件（禁止空 stub / 业务逻辑）
│   └── layout/                   # DashboardLayout · Navbar · OmniSearch · WS 指示
│
├── features/                     # 业务域（按产品模块）
│   ├── auth/
│   ├── quotes/ · data-center/
│   ├── screener/
│   ├── strategy/                 # IDE · workspace charts · sidebars
│   ├── trading/                  # OMS · backtest 入口壳 · risk · copilot 壳
│   ├── settings/ · system/
│   └── copilot/                  # AI 会话 UI（页内；全局抽屉见 FE-PROD-01）
│
├── hooks/ · stores/ · contexts/
├── lib/
│   └── api-client.ts             # HTTP 唯一入口（Fetch）
├── services/                     # WS / mock 等；禁止第二套 axios 客户端
├── types/ · workers/ · styles/ · locales/
```

## 硬约束

1. **布局**：线上唯一壳 = `components/layout/dashboard-layout.tsx` + `App.tsx` Outlet。
2. **HTTP**：只许 `@/lib/api-client`；禁止 axios / 平行 apiClient。
3. **包管理**：只提交 `pnpm-lock.yaml`；禁止 `package-lock.json`。
4. **单文件 ≤300 行**；债项见 `docs/TODO.md` FE-ARCH-02。
5. **新图表**：K 线 Lightweight-Charts；分析图 ECharts；禁止新增 recharts。

## 功能模块标准结构

```
features/<domain>/
├── index.tsx | <module>.tsx      # 路由懒加载入口
├── *-types.ts                    # 类型
├── components/ or workspace/     # 拆分后的子视图
└── stores/                       # 域内 Zustand（可选）
```
