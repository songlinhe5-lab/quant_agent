## 〇、OpenAPI / Swagger（BE-19）

| 项 | 说明 |
|:---|:---|

## 一、全局约定

### 1.1 Base URL

| 环境 | Base URL |
|:---|:---|
| 本地开发 | `http://localhost:8000` |
| 香港 VPS（生产） | `https://api.quant.yourdomain.com`（经 Cloudflare Tunnel） |

### 1.2 通用响应结构

所有 REST API 统一返回以下 JSON 结构，**严禁**各路由自定义格式：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {},
  "ts": 1719475200000
}
```

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `code` | `int` | `0` = 成功，非零 = 错误（见错误码表） |
| `msg` | `str` | 成功时为 `"ok"`，失败时为可读错误描述 |
| `data` | `any` | 业务数据负载，失败时可为 `null` |
| `ts` | `int` | 服务端 UTC 毫秒时间戳 |

### 1.3 鉴权方式

| 通道 | 鉴权机制 | Header |
|:---|:---|:---|
| 公开 REST API（`/api/v1/*`） | JWT Bearer Token | `Authorization: Bearer <access_token>` |
| WebSocket 行情（`/ws/v1/*`） | 连接时 QueryString | `?token=<access_token>` |
| SSE Agent 流（`/sse/v1/*`） | 连接时 QueryString | `?token=<access_token>` |
| 内部 Tool API（`/internal/v1/*`） | HMAC-SHA256 签名 | `X-Internal-Sig: <hmac_hex>` + `X-Internal-Ts: <utc_ms>` |

### 1.4 错误码表

| code | HTTP Status | 含义 | 处理建议 |
|:---|:---|:---|:---|
| `0` | `200` | 成功 | — |
| `1001` | `401` | Token 缺失 | 重新登录 |
| `1002` | `401` | Token 过期 | 用 Refresh Token 换新 Access Token |
| `1003` | `401` | Token 无效/篡改 | 清除本地凭证，重新登录 |
| `1004` | `403` | 权限不足 | 检查账户角色 |
| `1005` | `403` | HMAC 签名校验失败 | 检查内部密钥配置 |
| `2001` | `400` | 请求参数校验失败 | 查看 `msg` 字段获取具体字段错误 |
| `2002` | `404` | 资源不存在 | 检查资源 ID |
| `3001` | `503` | Futu OpenD 连接断开 | 等待自动重连或检查 OpenD 进程 |
| `3002` | `503` | Redis 不可用 | 检查 Redis 服务状态 |
| `3003` | `503` | 外部 API 熔断中 | 等待熔断器半开（约 60s） |
| `5000` | `500` | 内部未知错误 | 查看后端日志（携带 `trace_id`） |

### 1.5 分页约定

支持分页的接口统一使用 cursor-based 分页：

```json
// 请求
GET /api/v1/orders?cursor=<last_id>&limit=50

// 响应 data 结构
{
  "items": [...],
  "next_cursor": "ord_abc123",
  "has_more": true
}
```

## 二、认证接口（`/api/v1/auth`）

### 2.1 登录

```
POST /api/v1/auth/login
```

**Request Body:**
```json
{
  "password": "string"
}
```

> 个人系统无多用户，只验证主密码。

**Response `data`:**
```json
{
  "access_token": "eyJ...",
  "expires_in": 900,
  "token_type": "Bearer"
}
```

> Refresh Token 通过 `Set-Cookie: refresh_token=...; HttpOnly; Secure; SameSite=Strict` 写入浏览器，前端无需手动处理。

### 2.2 刷新 Token

```
POST /api/v1/auth/refresh
```

> 不需要 Request Body，自动读取 HttpOnly Cookie 中的 Refresh Token。

**Response `data`:** 同 2.1

### 2.3 登出

```
POST /api/v1/auth/logout
```

> 清除服务端 Refresh Token 黑名单记录，并通知客户端清除 Cookie。

## 六、Hermes Agent 接口（SSE）

### 6.1 发送问题并获取流式响应

```
POST /api/v1/chat
Content-Type: application/json
Accept: text/event-stream
```

> **V1.1 纠偏**：路径为 `/api/v1/chat`（不是 `/api/v1/agent/chat`）。

**Request Body:**
```json
{
  "message": "分析一下苹果公司最近的走势，给出买卖建议",
  "session_id": "sess_abc123",
  "context": {
    "current_symbol": "AAPL"
  }
}
```

**SSE Stream 格式：**
```
event: thinking
data: {"step": "tool_call", "tool": "get_broker_market_data", "args": {"action": "QUOTE", "symbol": "AAPL"}}

event: tool_result
data: {"tool": "get_broker_market_data", "result_summary": "AAPL 当前价 213.45，涨 0.58%"}

event: message
data: {"content": "## AAPL 走势分析\n\n根据最新数据...", "done": false}

event: message
data: {"content": "", "done": true, "session_id": "sess_abc123"}

event: error
data: {"code": 3001, "msg": "Futu OpenD 连接断开"}
```

| SSE Event 类型 | 含义 |
|:---|:---|
| `thinking` | Agent 推理步骤（Tool 调用计划） |
| `tool_result` | Tool 执行结果摘要 |
| `message` | 最终 Markdown 内容片段，`done=true` 标志结束 |
| `error` | 流中断错误 |

## 九、内部 Tool API

> Hermes Agent Tools **禁止**直连外部数据源；当前生产路径是调用公开 Gateway（`/api/v1/market/*` 等）或极少量 `/api/v1/internal/*`（health / cache）。  
> 旧表里的 `/internal/v1/*` 为设计期规划，**V1.1 以 `docs/openapi.json` 中 `Internal` tag 为准**。

### HMAC 签名生成规则

```python
import hmac, hashlib, time

def sign(secret: str, method: str, path: str, body: str = "") -> tuple[str, int]:
    ts = int(time.time() * 1000)
    payload = f"{method}\n{path}\n{ts}\n{body}"
    sig = hmac.new(secret.encode(), payload.encode(), hashlib.sha256).hexdigest()
    return sig, ts
```

请求时携带：
```
X-Internal-Sig: <sig>
X-Internal-Ts: <ts_ms>
```

服务端校验：时间戳偏差 > 30s 的请求直接拒绝（防重放）。

### 内部接口（现行）

| 端点 | 方法 | 说明 |
|:---|:---|:---|
| `/api/v1/internal/health` | `GET` | 内部健康探活 |
| `/api/v1/internal/cache/clear` | `POST` | 清理内部缓存 |

> Agent Tool 实际走 Gateway 公开能力（行情/选股/新闻等），不再维护独立的 `/internal/v1/market/*` 镜像表。完整清单见 `docs/openapi.json`。
